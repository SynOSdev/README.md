## 📖 Background

Counter-intelligence in cybersecurity means detecting and neutralizing an adversary's intelligence-gathering efforts. Defenders deploy honeypots (fake vulnerable services), canary tokens (tripwires that alert when accessed), and deception networks to detect intruders. Attackers who blunder into these traps reveal their presence, techniques, and sometimes their identity.

Key concepts:
- **Honeypot** — A decoy system designed to attract and detect unauthorized access
- **Canary token** — A digital tripwire that alerts when a resource is accessed, opened, or resolved
- **Deception network** — An infrastructure of fake assets designed to mislead adversaries
- **Counter-OSINT** — Techniques to reduce your own information exposure in public sources
- **Disinformation** — Deliberately false information designed to mislead intelligence efforts

## 🔬 Lab Instructions

### Phase 1: Honeypot Detection and Analysis

1. Understand common honeypot characteristics:
   ```
   Indicators of a honeypot:
   - Service banners that are too perfect or generic
   - Unusual port combinations (SSH + Telnet + RDP on same host)
   - Services that accept any credentials
   - Responses that are too fast or too slow for the claimed service
   - Known honeypot signatures in service fingerprints
   ```
2. Use nmap service fingerprinting to detect anomalies:
   ```bash
   # Scan a target and look for honeypot indicators
   nmap -sV --version-intensity 5 -p 22,23,80,443,3389,8080 target.lab.local
   ```
3. Analyze service banners for deception indicators:
   ```python
   import socket

   def probe_service(host, port, timeout=5):
       """Connect to a service and capture its banner."""
       try:
           s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
           s.settimeout(timeout)
           s.connect((host, port))
           s.send(b"HEAD / HTTP/1.0\r\n\r\n")
           banner = s.recv(1024).decode(errors="replace")
           s.close()
           return banner
       except (socket.error, socket.timeout):
           return None

   def detect_honeypot_indicators(banner):
       """Analyze a banner for common honeypot signatures."""
       indicators = []
       honeypot_sigs = [
           "kippo", "cowrie", "dionaea", "glastopf", "conpot",
           "honeyd", "artillery", "opencanary"
       ]
       if banner:
           banner_lower = banner.lower()
           for sig in honeypot_sigs:
               if sig in banner_lower:
                   indicators.append(f"Known honeypot signature: {sig}")
           # Check for suspiciously detailed banners
           if len(banner) > 500:
               indicators.append("Unusually verbose banner — possible honeypot")
       return indicators
   ```
4. Build a honeypot scoring matrix:
   - Score each service on deception likelihood (0-100)
   - Factor in banner analysis, response timing, port correlation, and fingerprint mismatches

### Phase 2: Canary Token Detection

1. Understand how canary tokens work:
   ```
   Canary Token Types:
   - URL tokens: Unique URLs that alert when visited
   - DNS tokens: Hostnames that alert when resolved
   - Document tokens: Office/PDF files with embedded callbacks
   - AWS key tokens: Fake AWS credentials that alert on use
   - Email tokens: Addresses that alert when emailed
   ```
2. Detect URL-based canary tokens:
   ```python
   import re
   from urllib.parse import urlparse

   def detect_url_canaries(urls):
       """Identify potential canary token URLs."""
       canary_indicators = []
       known_canary_domains = [
           "canarytokens.com", "canary.tools",
           "dockle.host", "burpcollaborator.net"
       ]
       for url in urls:
           parsed = urlparse(url)
           # Check for known canary domains
           for domain in known_canary_domains:
               if domain in parsed.netloc:
                   canary_indicators.append({
                       "url": url, "type": "known_canary_domain",
                       "confidence": "HIGH"
                   })
           # Check for suspiciously unique paths (long random strings)
           if re.search(r'/[a-z0-9]{20,}', parsed.path):
               canary_indicators.append({
                   "url": url, "type": "suspicious_unique_path",
                   "confidence": "MEDIUM"
               })
       return canary_indicators
   ```
3. Detect DNS-based canary tokens:
   ```bash
   # Inspect DNS queries for canary resolution patterns
   # Look for queries to known canary token domains
   grep -E "(canarytokens|canary\.tools)" /var/log/syslog 2>/dev/null
   ```
4. Detect document-embedded canary tokens:
   ```python
   import zipfile
   import re

   def scan_docx_for_canaries(filepath):
       """Scan an Office document for embedded canary callbacks."""
       findings = []
       try:
           with zipfile.ZipFile(filepath) as z:
               for name in z.namelist():
                   content = z.read(name).decode(errors="replace")
                   # Check for external URLs in relationships
                   urls = re.findall(r'https?://[^\s<>"]+', content)
                   for url in urls:
                       if any(d in url for d in ["canarytokens", "canary.tools"]):
                           findings.append(f"Canary token URL in {name}: {url}")
                       # Check for tracking pixels
                       if re.search(r'https?://.*\.(png|gif)\?.*token', url):
                           findings.append(f"Tracking pixel in {name}: {url}")
       except (zipfile.BadZipFile, FileNotFoundError):
           pass
       return findings
   ```

### Phase 4: Disinformation Awareness

1. Study common disinformation techniques used in cyber operations:
   ```
   Deception Patterns:
   - False flag operations: Planting artifacts that blame another group
   - Poisoned leaks: Mixing real data with fabricated information
   - Fabricated personas: Creating fake identities to spread narratives
   - Manipulated metadata: Altering timestamps, language settings, IP data
   - Decoy infrastructure: Setting up fake C2 servers to mislead investigators
   ```
2. Develop verification techniques:
   - Cross-reference claims across multiple independent sources
   - Verify metadata consistency (timezone vs. claimed location)
   - Check for signs of content manipulation (copy-paste artifacts, inconsistent formatting)
3. Document your analysis methodology as a repeatable process

#!/usr/bin/env python3
"""GRIMOIRE Counter-Intel Scanner - LAB-035 Build Your Own"""

import socket
import re
import json
import subprocess
import argparse
from datetime import datetime


class CounterIntelScanner:
    def __init__(self):
        self.findings = []

    def _add_finding(self, category, item, confidence, details):
        self.findings.append({
            "category": category,
            "item": item,
            "confidence": confidence,
            "details": details,
            "timestamp": datetime.utcnow().isoformat()
        })

    def scan_honeypots(self, host, ports):
        """Scan services for honeypot indicators."""
        honeypot_sigs = ["kippo", "cowrie", "dionaea", "glastopf",
                         "conpot", "honeyd", "opencanary", "artillery"]
        for port in ports:
            try:
                s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
                s.settimeout(5)
                s.connect((host, port))
                s.send(b"\r\n")
                banner = s.recv(1024).decode(errors="replace")
                s.close()

                for sig in honeypot_sigs:
                    if sig in banner.lower():
                        self._add_finding(
                            "honeypot", f"{host}:{port}",
                            "HIGH", f"Known signature: {sig}"
                        )
                        break
                else:
                    # TODO: Add timing analysis and banner anomaly detection
                    if len(banner) > 500:
                        self._add_finding(
                            "honeypot", f"{host}:{port}",
                            "LOW", "Verbose banner — review manually"
                        )
            except (socket.error, socket.timeout):
                continue

    def scan_canary_urls(self, urls):
        """Detect canary token URLs."""
        canary_domains = ["canarytokens.com", "canary.tools",
                          "burpcollaborator.net"]
        for url in urls:
            for domain in canary_domains:
                if domain in url:
                    self._add_finding("canary", url, "HIGH",
                                      f"Known canary domain: {domain}")
            if re.search(r'/[a-z0-9]{25,}', url):
                self._add_finding("canary", url, "MEDIUM",
                                  "Suspiciously long unique path segment")

    def scan_documents(self, filepath):

        try:
            with zipfile.ZipFile(filepath) as z:

                        self._add_finding("surveillance", line.strip(), "LOW",
                                          "Non-standard listening service")
        except FileNotFoundError:

    def export_report(self, path="counterintel_report.json"):
        """Export the detection report."""
        report = {
            "scan_time": datetime.utcnow().isoformat(),
            "total_findings": len(self.findings),

        for f in self.findings:
            cat = f["category"]

                report["findings_by_category"].get(cat, 0) + 1
