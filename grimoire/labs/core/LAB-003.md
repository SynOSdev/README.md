## 📖 Background

Linux administration is a core competency for cybersecurity professionals. Whether you are hardening a server, investigating an incident, or setting up a lab environment, system administration skills are essential. Understanding how Linux manages users, processes, services, and storage gives you the foundation to secure or test any system.

---

## 🔬 Lab Instructions

### Phase 1: User and Group Management

1. Create a new user with `useradd` and set a password
2. Create groups and assign users to them
3. Configure sudo access for specific users
4. Review `/etc/passwd`, `/etc/shadow`, and `/etc/group`
5. Lock and unlock user accounts

### Phase 2: Process and Service Management

1. View running processes with `ps aux` and `top`
2. Identify resource-intensive processes
3. Start, stop, enable, and disable services with `systemctl`
4. View service logs with `journalctl`
5. Kill and manage processes with `kill` and `nice`

### Phase 3: Storage and Scheduling

1. Check disk usage with `df -h` and `du -sh`
2. Understand mount points and `/etc/fstab`
3. Create a cron job that runs a maintenance script
4. Create a systemd timer as an alternative to cron
5. Verify scheduled tasks are executing correctly
