## 📋 Mission Briefing

The web is the largest attack surface in the world. Understanding how web technologies work — from HTTP requests to server-side processing to client-side rendering — is essential for both attacking and defending web applications. This lab breaks down the web stack so you can operate at every layer.

## 📖 Background

The modern web is built on HTTP/HTTPS. Understanding request methods (GET, POST, PUT, DELETE), status codes (200, 301, 403, 404, 500), headers, cookies, and authentication flows is critical for web security work. Whether you are testing a web app, investigating an incident, or building secure services, these fundamentals are essential.

---

## 🔬 Lab Instructions

### Phase 1: HTTP Fundamentals

1. Send requests with curl: `curl -v https://example.com`
2. Examine request and response headers
3. Test different HTTP methods: GET, POST, HEAD, OPTIONS
4. Understand status codes by triggering 200, 301, 404, 403, and 500 responses
5. Inspect cookies and set-cookie headers

### Phase 2: Web Application Mapping

1. Use `dirb` or `gobuster` to enumerate directories on a test server
2. Map out the application structure: pages, forms, API endpoints
3. Identify technologies used (server software, frameworks, languages)
4. Document all entry points where user input is accepted

### Phase 3: Traffic Inspection

1. Use browser DevTools Network tab to monitor all requests
2. Identify AJAX calls, API requests, and resource loading
3. Examine authentication flows (login, session tokens, cookies)
4. Look for sensitive data in transit
