## 📖 Background

Cryptography underpins virtually all modern security: HTTPS, VPNs, authentication, digital signatures, disk encryption, and secure communications. Understanding the fundamentals — hashing, symmetric encryption, asymmetric encryption, and digital signatures — is essential for every cybersecurity role.

Key concepts:
- **Hashing** — One-way function producing a fixed-length digest
- **Symmetric encryption** — Same key encrypts and decrypts (AES, ChaCha20)
- **Asymmetric encryption** — Key pairs where public encrypts, private decrypts (RSA, ECC)
- **Digital signatures** — Prove authenticity and integrity using private keys
- **Key management** — Secure generation, storage, and rotation of keys

## 🔬 Lab Instructions

### Phase 1: Hashing

1. Generate hashes of files: `sha256sum file.txt`, `md5sum file.txt`
2. Modify a single byte and observe how the hash changes completely
3. Verify file integrity by comparing hashes
4. Understand why MD5 and SHA-1 are deprecated for security purposes
5. Explore hash collisions conceptually

### Phase 2: Symmetric Encryption

1. Encrypt a file with OpenSSL: `openssl enc -aes-256-cbc -salt -in plain.txt -out encrypted.bin`
2. Decrypt the file and verify contents
3. Experiment with different algorithms (AES-128, ChaCha20)
4. Understand the importance of initialization vectors and salts

### Phase 3: Asymmetric Encryption and Signatures

1. Generate a GPG key pair: `gpg --full-generate-key`
2. Export and share your public key
3. Encrypt a message for another user using their public key
4. Create a digital signature: `gpg --detach-sign file.txt`
5. Verify a signature: `gpg --verify file.txt.sig file.txt`
6. Generate SSH key pairs with `ssh-keygen`
