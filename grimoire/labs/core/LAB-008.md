## 📖 Background

Vulnerability assessment is the process of identifying, quantifying, and prioritizing security weaknesses. It differs from penetration testing in that it focuses on discovery and reporting rather than exploitation. Understanding the vulnerability lifecycle — discovery, classification, prioritization, remediation, and verification — is essential for all cybersecurity roles.

Key concepts:
- **CVE** — Common Vulnerabilities and Exposures identifiers
- **CVSS** — Common Vulnerability Scoring System for severity rating
- **CWE** — Common Weakness Enumeration for vulnerability categories
- **Attack surface** — Total set of points where an attacker can interact
- **Risk assessment** — Likelihood × Impact evaluation framework

## 🔬 Lab Instructions

### Phase 1: Network Vulnerability Discovery

1. Run an nmap scan: `nmap -sV -sC -oN scan.txt target`
2. Identify open ports, services, and their versions
3. Look for known vulnerable versions using `searchsploit`
4. Document all findings with port, service, version, and potential vulnerabilities

### Phase 2: System Auditing

1. Run `lynis audit system` on the local system
2. Review the hardening index score
3. Examine each finding category: authentication, networking, storage, logging
4. Prioritize findings by severity and ease of remediation

### Phase 3: Reporting

1. Create a structured vulnerability report with:
   - Executive summary
   - Methodology description
   - Findings table (sorted by severity)
   - Remediation recommendations
   - Risk ratings for each finding
2. Use CVSS scoring to rate each vulnerability
3. Provide specific remediation steps for the top 5 findings
