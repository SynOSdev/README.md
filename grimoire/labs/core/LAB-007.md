## 📋 Mission Briefing

Logs are the memory of a system. Every action, connection, error, and event is recorded somewhere. The ability to find, parse, correlate, and analyze logs is what separates a skilled operator from a novice. This lab introduces log analysis fundamentals and the basics of Security Information and Event Management systems.

## 📖 Background

Logs are generated by virtually every system component: operating systems, applications, firewalls, authentication systems, web servers, and databases. A SIEM collects these logs, normalizes them into a common format, correlates events across sources, and generates alerts based on detection rules. Understanding log analysis is fundamental for incident response, threat hunting, and compliance.

---

## 🔬 Lab Instructions

### Phase 1: Log Discovery

1. Explore `/var/log/` and identify key log files
2. Read authentication logs: `journalctl -u ssh` or `/var/log/auth.log`
3. Read system logs: `journalctl` or `/var/log/syslog`
4. Identify log formats: syslog, JSON, CSV, custom formats

### Phase 2: Log Parsing and Analysis

1. Use `grep` to search for specific events (failed logins, errors)
2. Use `awk` to extract specific fields from log entries
3. Use `sort | uniq -c | sort -rn` to find top events
4. Build a timeline of events using timestamps
5. Correlate related events across log files

### Phase 3: Detection Rules

1. Identify patterns that indicate suspicious activity
2. Write grep/awk patterns that detect brute-force login attempts
3. Create a rule that detects unusual process execution
4. Document your rules with descriptions and severity levels
