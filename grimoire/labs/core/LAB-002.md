## 📖 Background

All digital communication occurs over networks using standardized protocols. Understanding how packets flow — from application data, through transport segmentation, to network routing and physical transmission — is essential for both attacking and defending systems.

Key concepts:
- **OSI Model** — 7-layer reference model for network communication
- **TCP/IP Stack** — Practical 4-layer model used in real networks
- **Packets** — Units of data with headers containing routing and protocol information
- **Ports** — Logical endpoints that identify specific services
- **DNS** — Domain Name System translating human names to IP addresses

## 🔬 Lab Instructions

### Phase 1: Network Reconnaissance

1. Use `ip addr` or `ifconfig` to identify your network interfaces
2. Use `ss -tulnp` to view active listening ports and services
3. Run `ping` and `traceroute` to map network paths
4. Use `dig` or `nslookup` to query DNS records

### Phase 2: Packet Capture

1. Start a capture with `tcpdump -i eth0 -w capture.pcap`
2. Generate traffic by browsing, pinging, and running DNS queries
3. Stop the capture and examine the pcap file
4. Filter captures: `tcpdump -r capture.pcap 'tcp port 80'`
5. View packet details: `tcpdump -r capture.pcap -X -c 10`

### Phase 3: Traffic Analysis

1. Open the capture in tshark: `tshark -r capture.pcap`
2. Apply display filters to isolate specific protocols
3. Follow TCP streams to reconstruct conversations
4. Identify DNS queries and their responses
5. Map out all unique IP addresses and the services they communicated with
