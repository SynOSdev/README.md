## 📖 Background

The terminal is the primary interface for cybersecurity professionals. While GUI tools exist, the command line provides speed, precision, scriptability, and access to systems where no graphical interface is available. Mastering shell navigation is the foundation upon which all other GRIMOIRE skills are built.

Key concepts:
- **Shell** — The command interpreter (bash, zsh, sh)
- **Filesystem hierarchy** — The directory tree structure in Unix/Linux
- **Standard streams** — stdin, stdout, stderr and how to redirect them
- **Pipes** — Connecting command output to another command's input
- **Permissions** — Read, write, execute for owner, group, others

## 🔬 Lab Instructions

### Phase 1: Filesystem Navigation

1. Open your terminal in the synOS environment
2. Use `pwd` to display your current working directory
3. Use `ls -la` to list all files including hidden ones
4. Navigate to `/etc` using `cd` and explore system configuration files
5. Use `tree -L 2` to visualize directory structures
6. Practice using relative paths (`../`, `./`) and absolute paths

### Phase 2: File Operations

1. Create a working directory: `mkdir -p ~/grimoire/lab-001/workspace`
2. Create files using `touch`, `echo`, and `cat`
3. Copy files with `cp`, move with `mv`, remove with `rm`
4. Use `find` to locate files by name, type, and modification time
5. Search file contents with `grep` and `grep -r`

### Phase 3: Pipes and Redirects

1. Chain commands using pipes: `ls -la | grep ".conf"`
2. Redirect output to files: `echo "data" > output.txt`
3. Append to files: `echo "more data" >> output.txt`
4. Redirect errors: `command 2> errors.log`
5. Combine stdout and stderr: `command &> all_output.log`
6. Use `tee` to split output to screen and file simultaneously

### Phase 4: Permissions

1. View permissions with `ls -la`
2. Change permissions using `chmod` (symbolic and octal notation)
3. Change ownership with `chown`
4. Understand the permission model: user/group/other, read/write/execute
5. Create a file, restrict its permissions, and verify access control
