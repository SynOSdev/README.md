## 📖 Background

Security dashboards aggregate data from multiple sources into a single view, giving analysts situational awareness at a glance. Commercial SIEM dashboards can cost thousands, but the fundamentals are straightforward: collect data, format it, and display it. Understanding how dashboards work under the hood makes you a better analyst — you know what data is real, what is delayed, and what might be missing.

Key concepts:
- **Metric collection** — Polling system stats at regular intervals
- **Log parsing** — Extracting structured fields from raw log lines
- **Data serialization** — Converting metrics to JSON for API consumption
- **Server-sent events** — Pushing real-time updates to the browser without polling
- **Dashboard layout** — Organizing panels for rapid situational awareness

## 🔬 Lab Instructions

### Phase 1: Metric Collection Engine

1. Install the required libraries: `pip install flask psutil`
2. Create a metric collector module that gathers:
   - CPU usage percentage (per-core and aggregate)
   - Memory usage (total, used, available, percentage)
   - Disk usage per mount point
   - Network I/O counters (bytes sent/received, packets, errors)
   - Active network connections count by state
3. Store metrics in a time-series list with timestamps
4. Implement a rolling window that retains the last 60 data points
5. Test the collector by printing JSON output to the terminal

### Phase 2: Log Ingestion and Parsing

1. Create a log watcher that monitors one or more log files (e.g., `/var/log/syslog`, `/var/log/auth.log`, or your own test logs)
2. Parse each log line into structured fields: timestamp, source, severity, message
3. Implement severity classification using keyword matching:
   - **CRITICAL**: `error`, `fatal`, `emergency`
   - **WARNING**: `warn`, `fail`, `denied`
   - **INFO**: everything else
4. Store the last 200 parsed log entries in a ring buffer
5. Test with sample log files to verify parsing accuracy

### Phase 3: Dashboard Web Application

1. Create a Flask application with routes for:
   - `GET /` — Main dashboard page
   - `GET /api/metrics` — Current system metrics as JSON
   - `GET /api/logs` — Recent parsed log entries as JSON
   - `GET /api/alerts` — Filtered high-severity log entries
   - `GET /api/status` — System health summary
2. Build an HTML dashboard template with panels for:
   - System metrics gauges (CPU, memory, disk)
   - Network traffic line chart
   - Live log feed with severity color coding
   - Alert panel showing only WARNING and CRITICAL entries
   - System status indicators (green/yellow/red)
3. Add JavaScript to fetch API endpoints every 5 seconds and update panels
4. Style the dashboard with a dark theme suitable for SOC environments
5. Start the server and verify all panels display live data
