## 📖 Background

Real security operations platforms are not single tools — they are integrated suites where data flows between modules: scanners feed detection pipelines, detection pipelines trigger dashboards, dashboards inform responders, and secure channels protect everything in transit. Building this integration yourself is the ultimate test of your engineering and security skills. It also produces something genuinely useful — a personal security platform tailored to your needs.

Key concepts:
- **Service-oriented architecture** — Each tool runs as an independent module with a defined interface
- **Event bus** — A shared communication channel for inter-module messages
- **Configuration management** — Centralized settings that all modules read from
- **Process management** — Starting, stopping, and monitoring module health
- **Integration testing** — Verifying that data flows correctly between all modules

   - Dashboard: host, port, refresh interval

   - Scanner: default targets, port ranges, scan profiles
   - IR Toolkit: evidence output path, collection profiles
   - Secure Comms: listen port, encryption settings
3. Implement a configuration loader that all modules import

### Phase 2: Event Bus and Module Integration

1. Build an event bus using Python's `multiprocessing.Queue`:
   - Each module can publish events (alerts, scan results, status updates)
   - Each module can subscribe to event types it cares about
   - Events carry a standard envelope: `{type, source, timestamp, payload}`
2. Wire the integrations:
   - **Scanner → Detection**: Scan results are published as events; the detection pipeline has rules that trigger on scan findings
   - **Detection → Dashboard**: Alerts are published to the event bus; the dashboard API reads from the alert queue
   - **Detection → IR Toolkit**: High-severity alerts can trigger automatic evidence collection
   - **All modules → Secure Comms**: Sensitive inter-module messages are encrypted
3. Test each integration point with synthetic events

### Phase 3: Central CLI and Process Management

1. Build a CLI using `argparse` with subcommands:
   - `suite.py start [module|all]` — Start one or all modules
   - `suite.py stop [module|all]` — Stop running modules
   - `suite.py status` — Display health of all modules
   - `suite.py scan <target>` — Run a vulnerability scan
   - `suite.py respond <alert-id>` — Launch IR collection for an alert
   - `suite.py report` — Generate the comprehensive post-game report
2. Implement process management:
   - Each module runs as a managed subprocess
   - Health checks verify each module is responsive
   - Automatic restart on module failure
3. Build the post-game report generator:
   - Summarize all scans performed, alerts generated, incidents responded to
   - Include a skills matrix showing which GRIMOIRE skills were exercised
   - List all tools built and their capabilities
   - Output as both text and HTML

def setup_logger(name, level=logging.INFO):
    """Create a structured logger for a module."""
    logger = logging.getLogger(name)

    handler = logging.StreamHandler()
    fmt = logging.Formatter(f"[%(asctime)s][{name}][%(levelname)s] %(message)s",

    handler.setFormatter(fmt)

class EventBus:
    """Inter-module communication via multiprocessing queues."""

    def __init__(self):
        self._queues = {}  # event_type -> list of subscriber queues

    def subscribe(self, event_type):
        """Subscribe to an event type. Returns a Queue to read from."""
        q = Queue()
        self._queues.setdefault(event_type, []).append(q)
        return q

    def publish(self, event_type, source, payload):
        """Publish an event to all subscribers of that type."""
        event = {
            "type": event_type,
            "source": source,
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "payload": payload,
        }
        for q in self._queues.get(event_type, []):
            q.put(event)
        for q in self._queues.get("*", []):  # wildcard subscribers
            q.put(event)

# --- Process Manager ---
class ProcessManager:
    """Manage suite module subprocesses."""

    def __init__(self):
        self.processes = {}

    def start_module(self, name, target, args=()):
        """Start a module as a subprocess."""
        if name in self.processes and self.processes[name].is_alive():
            print(f"[manager] {name} is already running.")
            return
        p = Process(target=target, args=args, name=name, daemon=True)
        p.start()
        self.processes[name] = p
        print(f"[manager] Started {name} (PID {p.pid})")

    def stop_module(self, name):
        """Stop a running module."""
        if name in self.processes and self.processes[name].is_alive():
            self.processes[name].terminate()
            self.processes[name].join(timeout=5)
            print(f"[manager] Stopped {name}")
        else:
            print(f"[manager] {name} is not running.")

    def status(self):
        """Display status of all modules."""
        print(f"\n{'Module':<20} {'PID':<10} {'Status':<10}")
        print("-" * 40)
        for name, proc in self.processes.items():
            status = "RUNNING" if proc.is_alive() else "STOPPED"
            print(f"{name:<20} {proc.pid:<10} {status:<10}")
        if not self.processes:
            print("  No modules registered.")
        print()

    def stop_all(self):
        for name in list(self.processes.keys()):
            self.stop_module(name)
