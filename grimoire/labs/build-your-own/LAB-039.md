## 📖 Background

Vulnerability scanners combine several techniques: they discover open ports, identify running services, determine software versions, and then look up known vulnerabilities for those versions. Each stage is a distinct skill. By building each stage yourself, you gain the ability to customize, extend, and troubleshoot scanning — and you deeply understand the data that commercial scanners produce.

Key concepts:
- **Port scanning** — Probing TCP ports to identify open services
- **Banner grabbing** — Reading the initial data a service sends on connection
- **Service fingerprinting** — Matching banners to known service signatures
- **CVE lookup** — Querying the National Vulnerability Database by CPE or keyword
- **Report generation** — Compiling findings into actionable output

> **⚠️ Important:** Only scan systems you own or have explicit written authorization to test. Unauthorized scanning is illegal.

## 🔬 Lab Instructions

### Phase 1: Port Scanner

1. Create a `PortScanner` class that:
   - Accepts a target IP and a port range (e.g., 1–1024)
   - Attempts TCP connections with a configurable timeout
   - Uses `concurrent.futures.ThreadPoolExecutor` for parallel scanning
   - Returns a list of open ports with response times
2. Implement common port range presets: top-100, top-1000, full-65535
3. Add basic rate limiting to avoid overwhelming the target
4. Test against a local service (e.g., start a Python HTTP server on a known port)

### Phase 2: Service Detection and Fingerprinting

1. Create a `ServiceDetector` class that:
   - Connects to each open port and reads the initial banner (first 1024 bytes)
   - Sends protocol-specific probes for ports that do not send banners automatically (HTTP HEAD, SMTP EHLO, etc.)
   - Matches banners against a signature dictionary to identify the service and version
2. Build a signature dictionary with at least 10 common services:
   - SSH, HTTP/HTTPS, FTP, SMTP, MySQL, PostgreSQL, Redis, DNS, IMAP, POP3
3. Extract version strings using regex patterns
4. Test against local services and verify correct identification

### Phase 3: CVE Lookup and Report Generation

1. Create a `CVELookup` class that:
   - Takes a service name and version string
   - Queries the public NVD API (`https://services.nvd.nist.gov/rest/json/cves/2.0`) by keyword
   - Parses results to extract CVE ID, description, CVSS score, and severity
   - Caches results to avoid repeated API calls for the same query
2. Create a `ReportGenerator` class that:
   - Takes the full scan results (open ports, services, versions, CVEs)
   - Renders an HTML report using Jinja2 templates
   - Includes a summary table, per-port findings, and CVE details
   - Color-codes findings by CVSS severity
3. Run a full scan against a test target and generate the report

class PortScanner:
    """TCP port scanner with parallel execution."""

    TOP_100 = [21,22,23,25,53,80,110,111,135,139,143,443,445,993,995,
               1723,3306,3389,5432,5900,6379,8080,8443,8888,27017]

    def __init__(self, target, ports=None, timeout=1.0, workers=50):
        self.target = str(ip_address(target))
        self.ports = ports or self.TOP_100
        self.timeout = timeout
        self.workers = workers

    def scan_port(self, port):
        """Attempt a TCP connection to a single port."""
        try:
            with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
                s.settimeout(self.timeout)
                start = datetime.utcnow()
                result = s.connect_ex((self.target, port))
                elapsed = (datetime.utcnow() - start).total_seconds()
                if result == 0:
                    return {"port": port, "state": "open", "response_ms": round(elapsed * 1000)}
        except Exception:
            pass
        return None

    def run(self):
        """Scan all configured ports in parallel."""
        results = []
        with ThreadPoolExecutor(max_workers=self.workers) as pool:
            futures = {pool.submit(self.scan_port, p): p for p in self.ports}
            for future in as_completed(futures):
                result = future.result()
                if result:
                    results.append(result)
        return sorted(results, key=lambda r: r["port"])
