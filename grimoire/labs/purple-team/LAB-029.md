## 📖 Background

Adversary emulation goes beyond automated scanning. It replicates the decision-making, tooling, and sequencing of a real threat actor. Organizations like MITRE publish detailed profiles of APT groups along with the ATT&CK techniques each group uses. By emulating these groups, you test your defenses against realistic, intelligence-driven attack scenarios.

Key concepts:
- **Threat Actor Profile** — A documented description of an adversary's goals, targets, and TTPs
- **Emulation Plan** — A sequenced set of actions mirroring a specific threat actor's operations
- **Kill Chain Phase** — A stage in the attack lifecycle (recon, delivery, exploitation, etc.)
- **CALDERA** — MITRE's open-source adversary emulation platform (conceptual reference)
- **Detection Delta** — The difference between what you can detect and what the emulated adversary does

## 🔬 Lab Instructions

### Phase 1: Threat Actor Research

1. Choose a well-documented threat group from ATT&CK:
   ```bash
   mkdir -p ~/emulation && cd ~/emulation
   # Download ATT&CK data
   curl -sL https://raw.githubusercontent.com/mitre/cti/master/enterprise-attack/enterprise-attack.json \
     -o enterprise-attack.json 2>/dev/null || echo "Using cached data"
   # List threat groups
   jq -r '.objects[] | select(.type=="intrusion-set") | "\(.name) — \(.description // "" | .[0:80])"' \
     enterprise-attack.json 2>/dev/null | head -15
   ```
2. Select a group and extract their techniques:
   ```bash
   # Example: APT29 (Cozy Bear)
   GROUP="APT29"
   echo "=== $GROUP Technique Profile ==="
   # In practice, query the relationships in the STIX data
   # Simplified: known APT29 techniques
   cat > ~/emulation/apt29_techniques.json << 'EOF'
   [
     {"id": "T1566.001", "name": "Spearphishing Attachment", "tactic": "initial-access"},
     {"id": "T1059.001", "name": "PowerShell", "tactic": "execution"},
     {"id": "T1053.005", "name": "Scheduled Task", "tactic": "persistence"},
     {"id": "T1055",     "name": "Process Injection", "tactic": "defense-evasion"},
     {"id": "T1003.001", "name": "LSASS Memory", "tactic": "credential-access"},
     {"id": "T1018",     "name": "Remote System Discovery", "tactic": "discovery"},
     {"id": "T1021.002", "name": "SMB/Windows Admin Shares", "tactic": "lateral-movement"},
     {"id": "T1560.001", "name": "Archive via Utility", "tactic": "collection"},
     {"id": "T1041",     "name": "Exfiltration Over C2 Channel", "tactic": "exfiltration"}
   ]
   EOF
   jq -r '.[] | "\(.tactic): \(.id) — \(.name)"' ~/emulation/apt29_techniques.json
   ```

### Phase 2: Building the Emulation Plan

1. Create a phased emulation plan document:
   ```bash
   cat > ~/emulation/emulation_plan.yaml << 'PLAN'
   emulation_plan:
     name: "APT29 Emulation — Operation Cozy Duke"
     threat_actor: APT29
     objective: "Simulate APT29 intrusion lifecycle from phishing to exfiltration"
     phases:
       - phase: 1
         name: "Initial Access"
         technique: "T1566.001"
         action: "Send test phishing email with macro-enabled document"
         test_cmd: "echo 'Simulated phishing payload delivered' > /tmp/phish_payload.txt"
         detection_expected: "Email gateway alert, endpoint macro execution alert"
       - phase: 2
         name: "Execution"
         technique: "T1059.001"
         action: "Execute encoded PowerShell/Bash command"
         test_cmd: "bash -c 'echo GRIMOIRE_APT29_EXEC | base64'"
         detection_expected: "Process creation with encoded arguments"
       - phase: 3
         name: "Persistence"
         technique: "T1053.005"
         action: "Create scheduled task for persistence"
         test_cmd: "echo '*/5 * * * * echo persist' > /tmp/art_cron && rm /tmp/art_cron"
         detection_expected: "Cron/scheduled task creation alert"
       - phase: 4
         name: "Credential Access"
         technique: "T1003.001"
         action: "Simulate credential dumping"
         test_cmd: "echo 'Simulated LSASS dump — no real credentials accessed'"
         detection_expected: "LSASS access alert or memory dump detection"
       - phase: 5
         name: "Discovery"
         technique: "T1018"
         action: "Enumerate remote systems"
         test_cmd: "hostname && whoami && ip addr show lo"
         detection_expected: "Reconnaissance command sequence alert"
       - phase: 6
         name: "Exfiltration"
         technique: "T1041"
         action: "Exfiltrate data over simulated C2"
         test_cmd: "tar czf /tmp/exfil.tar.gz /etc/hostname 2>/dev/null && rm /tmp/exfil.tar.gz"
         detection_expected: "Unusual archive creation followed by outbound transfer"
   PLAN
   echo "Emulation plan created: ~/emulation/emulation_plan.yaml"
   ```

### Phase 3: Executing and Measuring

1. Run each phase and record detection results:
   ```bash
   echo "phase,technique,action,detected" > ~/emulation/results.csv
   # Phase 1
   echo 'Simulated phishing payload delivered' > /tmp/phish_payload.txt
   echo "1,T1566.001,phishing_delivery,false" >> ~/emulation/results.csv
   rm -f /tmp/phish_payload.txt
   # Phase 2
   bash -c 'echo GRIMOIRE_APT29_EXEC | base64' > /dev/null
   echo "2,T1059.001,encoded_execution,true" >> ~/emulation/results.csv
   # Continue for all phases...
   ```
2. Generate the coverage summary:
   ```bash
   total=$(tail -n +2 ~/emulation/results.csv | wc -l)
   detected=$(grep -c ",true$" ~/emulation/results.csv)
   echo "APT29 Emulation Coverage: $detected / $total phases detected"
   ```
3. Export Navigator layer:
   ```bash
   echo '{"name":"APT29 Emulation","versions":{"attack":"14","navigator":"4.9","layer":"4.5"},"domain":"enterprise-attack","techniques":[' > ~/emulation/apt29_layer.json
   first=true
   while IFS=',' read -r phase tid action det; do
     [ "$phase" = "phase" ] && continue
     color="#00ff00"; score=1
     [ "$det" = "false" ] && color="#ff0000" && score=0
     $first || echo "," >> ~/emulation/apt29_layer.json
     echo "{\"techniqueID\":\"$tid\",\"score\":$score,\"color\":\"$color\"}" >> ~/emulation/apt29_layer.json
     first=false
   done < ~/emulation/results.csv
   echo ']}'  >> ~/emulation/apt29_layer.json
   echo "Navigator layer: ~/emulation/apt29_layer.json"
   ```

## 🛠️ Build Your Own

**Build Objective:** Create a custom adversary emulation plan builder that generates step-by-step attack playbooks from threat intel

### Requirements
- Accept a threat actor name and retrieve their known techniques
- Generate a phased emulation plan with executable test commands
- Execute the plan and measure detection at each phase
- Produce a detection coverage report and Navigator layer

### Starter Framework
```python
#!/usr/bin/env python3
"""GRIMOIRE Adversary Emulation Builder — LAB-029 Build Your Own"""

import json
import subprocess
import os
from datetime import datetime, timezone

EMULATION_DIR = os.path.expanduser("~/emulation")
os.makedirs(EMULATION_DIR, exist_ok=True)

# Threat actor profiles with techniques and simulated test commands
ACTOR_PROFILES = {
    "APT29": {
        "description": "Russian state-sponsored group targeting government and diplomatic entities",
        "phases": [
            {"technique": "T1566.001", "tactic": "Initial Access",
             "action": "Spearphishing attachment delivery",
             "test_cmd": "echo 'SIM:phishing_payload' > /tmp/emu_phish && rm /tmp/emu_phish"},
            {"technique": "T1059.001", "tactic": "Execution",
             "action": "PowerShell/shell encoded command",
             "test_cmd": "echo 'GRIMOIRE_EMU_EXEC' | base64 > /dev/null"},
            {"technique": "T1053.005", "tactic": "Persistence",
             "action": "Scheduled task creation",
             "test_cmd": "echo '* * * * * echo emu' > /tmp/emu_cron && rm /tmp/emu_cron"},
            {"technique": "T1003.001", "tactic": "Credential Access",
             "action": "Credential dumping simulation",
             "test_cmd": "echo 'SIM:lsass_dump' > /dev/null"},
            {"technique": "T1018", "tactic": "Discovery",
             "action": "Remote system discovery",
             "test_cmd": "hostname && whoami && id"},
            {"technique": "T1041", "tactic": "Exfiltration",
             "action": "Data exfiltration over C2",
             "test_cmd": "tar czf /tmp/emu_exfil.tar.gz /etc/hostname 2>/dev/null; rm -f /tmp/emu_exfil.tar.gz"},
        ],
    },
    "APT28": {
        "description": "Russian military intelligence group targeting defense and media",
        "phases": [
            {"technique": "T1190", "tactic": "Initial Access",
             "action": "Exploit public-facing application",
             "test_cmd": "curl -s --max-time 2 http://127.0.0.1:9999/ -o /dev/null 2>/dev/null || true"},
            {"technique": "T1059.003", "tactic": "Execution",
             "action": "Windows Command Shell (simulated)",
             "test_cmd": "echo 'SIM:cmd_exec whoami /all' > /dev/null"},
            {"technique": "T1071.001", "tactic": "Command and Control",
             "action": "Web protocol C2 communication",
             "test_cmd": "curl -s --max-time 2 http://example.com -o /dev/null 2>/dev/null || true"},
        ],
    },
}

def build_plan(actor_name: str) -> dict:
    """Build an emulation plan for the given threat actor."""
    profile = ACTOR_PROFILES.get(actor_name)
    if not profile:
        print(f"Unknown actor: {actor_name}. Available: {list(ACTOR_PROFILES.keys())}")
        return {}
    plan = {
        "actor": actor_name,
        "description": profile["description"],
        "created_at": datetime.now(timezone.utc).isoformat(),
        "phases": [],
    }
    for i, phase in enumerate(profile["phases"], 1):
        plan["phases"].append({
            "phase_number": i,
            "technique": phase["technique"],
            "tactic": phase["tactic"],
            "action": phase["action"],
            "test_cmd": phase["test_cmd"],
            "detected": None,
        })
    return plan

def execute_plan(plan: dict) -> dict:
    """Run each phase and check for detection."""
    print(f"\n=== Executing emulation plan for {plan['actor']} ===\n")
    for phase in plan["phases"]:
        print(f"  Phase {phase['phase_number']}: {phase['tactic']} — {phase['action']}")
        try:
            subprocess.run(phase["test_cmd"], shell=True, timeout=10, capture_output=True)
        except subprocess.TimeoutExpired:
            pass
        # TODO: Replace with real SIEM query to check for detection
        phase["detected"] = False
        status = "DETECTED" if phase["detected"] else "NOT DETECTED"
        print(f"    [{status}] {phase['technique']}")
    return plan

def generate_report(plan: dict):
    """Produce emulation report and Navigator layer."""
    total = len(plan["phases"])
    detected = sum(1 for p in plan["phases"] if p["detected"])

    print(f"\n{'='*50}")
    print(f"  Emulation Report: {plan['actor']}")
    print(f"  Coverage: {detected}/{total} phases detected")
    print(f"{'='*50}")
    for p in plan["phases"]:
        mark = "✓" if p["detected"] else "✗"
        print(f"  [{mark}] Phase {p['phase_number']}: {p['technique']} — {p['tactic']}")

    # Navigator layer
    layer = {
        "name": f"{plan['actor']} Emulation Coverage",
        "versions": {"attack": "14", "navigator": "4.9", "layer": "4.5"},
        "domain": "enterprise-attack",
        "techniques": [
            {
                "techniqueID": p["technique"],
                "score": 1 if p["detected"] else 0,
                "color": "#00ff00" if p["detected"] else "#ff0000",
            }
            for p in plan["phases"]
        ],
    }
    layer_path = os.path.join(EMULATION_DIR, f"{plan['actor']}_layer.json")
    with open(layer_path, "w") as f:
        json.dump(layer, f, indent=2)
    print(f"\n[+] Navigator layer: {layer_path}")

    # Full report
    report_path = os.path.join(EMULATION_DIR, f"{plan['actor']}_report.json")
    with open(report_path, "w") as f:
        json.dump(plan, f, indent=2)
    print(f"[+] Full report: {report_path}")

if __name__ == "__main__":
    import sys
    actor = sys.argv[1] if len(sys.argv) > 1 else "APT29"
    plan = build_plan(actor)
    if plan:
        plan = execute_plan(plan)
        generate_report(plan)
```

- [ ] Detection status is recorded for every phase
