## 📖 Background

The MITRE ATT&CK framework organizes adversary behavior into a hierarchy. At the top are **Tactics** — the "why" of an attack (e.g., Initial Access, Persistence). Under each tactic are **Techniques** — the "how" (e.g., Phishing, Scheduled Task). Many techniques have **Sub-techniques** for finer granularity.

Key concepts:
- **Tactic** — The adversary's goal for a phase of the attack (14 Enterprise tactics)
- **Technique / Sub-technique** — The specific method used to achieve a tactic
- **Procedure** — A documented instance of a technique used by a specific threat actor or malware
- **Data Source** — The telemetry required to observe a technique (e.g., process creation, network traffic)
- **Mitigation** — A defensive action that reduces the effectiveness of a technique

## 🔬 Lab Instructions

### Phase 1: Exploring the ATT&CK Matrix

1. Open the ATT&CK Enterprise matrix at https://attack.mitre.org/matrices/enterprise/
2. Identify the 14 tactics and note how they flow left to right as a kill chain.
3. Download the ATT&CK STIX bundle for local querying:
   ```bash
   mkdir -p ~/attack_data && cd ~/attack_data
   curl -sL https://raw.githubusercontent.com/mitre/cti/master/enterprise-attack/enterprise-attack.json \
     -o enterprise-attack.json
   echo "Techniques: $(jq '[.objects[] | select(.type=="attack-pattern")] | length' enterprise-attack.json)"
   ```
4. List all tactics:
   ```bash
   jq -r '.objects[] | select(.type=="x-mitre-tactic") | .name' enterprise-attack.json | sort
   ```

### Phase 2: Mapping Activity to Techniques

1. Given this simulated alert log, identify the ATT&CK techniques:
   ```
   Alert 1: PowerShell executed encoded command — Base64 string decoded to download cradle
   Alert 2: New scheduled task created under SYSTEM context at 03:00 daily
   Alert 3: LSASS memory dumped via unsigned binary from Temp directory
   Alert 4: Large outbound HTTPS transfer to newly registered domain
   ```
2. Research each technique using the STIX data:
   ```bash
   # Example: find technique for "PowerShell"
   jq -r '.objects[] | select(.type=="attack-pattern") |
     select(.name | test("PowerShell";"i")) |
     "\(.external_references[0].external_id) — \(.name)"' enterprise-attack.json
   ```
3. Document your mappings in a CSV:
   ```bash
   cat > ~/alert_mappings.csv << 'EOF'
   alert_id,description,technique_id,technique_name,tactic
   1,PowerShell encoded command,T1059.001,PowerShell,Execution

   3,LSASS memory dump,T1003.001,LSASS Memory,Credential Access
   ```

## 🛠️ Build Your Own

**Build Objective:** Create a custom ATT&CK technique tracker that maps observed activity to framework techniques

### Requirements
- Accept alert descriptions or indicators as input
- Search the ATT&CK STIX data for matching techniques
- Store mappings in a local database (JSON file)
- Generate an ATT&CK Navigator layer from stored mappings
- Provide a summary of coverage by tactic

### Starter Framework
```python
#!/usr/bin/env python3
"""GRIMOIRE ATT&CK Technique Tracker — LAB-026 Build Your Own"""

import json
import os
import re
import sys

ATTACK_DATA = os.path.expanduser("~/attack_data/enterprise-attack.json")
MAPPINGS_DB = os.path.expanduser("~/attack_mappings.json")

def load_attack_data() -> list:
    """Load ATT&CK techniques from the STIX bundle."""
    with open(ATTACK_DATA) as f:
        data = json.load(f)
    techniques = []
    for obj in data.get("objects", []):
        if obj.get("type") == "attack-pattern" and not obj.get("revoked"):
            tid = ""
            for ref in obj.get("external_references", []):
                if ref.get("source_name") == "mitre-attack":
                    tid = ref.get("external_id", "")
                    break
            tactics = [
                p.get("phase_name", "")
                for p in obj.get("kill_chain_phases", [])
            ]
            techniques.append({
                "id": tid,
                "name": obj.get("name", ""),
                "description": obj.get("description", "")[:200],
                "tactics": tactics,
            })
    return techniques

def search_techniques(query: str, techniques: list) -> list:
    """Search techniques by keyword in name or description."""
    pattern = re.compile(re.escape(query), re.IGNORECASE)
    return [t for t in techniques if pattern.search(t["name"]) or pattern.search(t["description"])]

def save_mapping(alert_id: str, technique_id: str, notes: str):
    """Persist an alert-to-technique mapping."""
    mappings = []
    if os.path.exists(MAPPINGS_DB):
        with open(MAPPINGS_DB) as f:
            mappings = json.load(f)
    mappings.append({"alert_id": alert_id, "technique_id": technique_id, "notes": notes})
    with open(MAPPINGS_DB, "w") as f:
        json.dump(mappings, f, indent=2)
    print(f"[+] Mapped alert {alert_id} -> {technique_id}")

def coverage_summary(techniques: list):
    """Print coverage by tactic."""
    if not os.path.exists(MAPPINGS_DB):
        print("No mappings yet.")
        return
    with open(MAPPINGS_DB) as f:
        mappings = json.load(f)
    mapped_ids = {m["technique_id"] for m in mappings}
    tactic_counts = {}
    for t in techniques:
        for tac in t["tactics"]:
            tactic_counts.setdefault(tac, {"total": 0, "covered": 0})
            tactic_counts[tac]["total"] += 1
            if t["id"] in mapped_ids:
                tactic_counts[tac]["covered"] += 1
    print(f"\n{'Tactic':<30} {'Covered':>8} {'Total':>8} {'%':>6}")
    print("-" * 56)
    for tac, counts in sorted(tactic_counts.items()):
        pct = counts["covered"] * 100 // counts["total"] if counts["total"] else 0
        print(f"{tac:<30} {counts['covered']:>8} {counts['total']:>8} {pct:>5}%")

def generate_layer():
    """Export an ATT&CK Navigator layer."""
    if not os.path.exists(MAPPINGS_DB):
        print("No mappings to export.")
        return
    with open(MAPPINGS_DB) as f:
        mappings = json.load(f)
    layer = {
        "name": "GRIMOIRE Technique Tracker",
        "versions": {"attack": "14", "navigator": "4.9", "layer": "4.5"},
        "domain": "enterprise-attack",
        "techniques": [
            {"techniqueID": m["technique_id"], "score": 1, "color": "#00ff00"}
            for m in mappings
        ],
    }
    path = os.path.expanduser("~/tracker_layer.json")
    with open(path, "w") as f:
        json.dump(layer, f, indent=2)
    print(f"[+] Navigator layer saved to {path}")

if __name__ == "__main__":
    techs = load_attack_data()
    if len(sys.argv) < 2:
        print("Usage: tracker.py {search|map|summary|export} [args]")
        sys.exit(1)
    cmd = sys.argv[1]
    if cmd == "search" and len(sys.argv) > 2:
        for t in search_techniques(sys.argv[2], techs):
            print(f"  {t['id']:<15} {t['name']}")
    elif cmd == "map" and len(sys.argv) > 3:
        save_mapping(sys.argv[2], sys.argv[3], " ".join(sys.argv[4:]))
    elif cmd == "summary":
        coverage_summary(techs)
    elif cmd == "export":
        generate_layer()
    else:
        print("Usage: tracker.py {search|map|summary|export} [args]")
```

### Validation
- [ ] `search` returns matching ATT&CK techniques for a keyword
- [ ] `map` persists alert-to-technique mappings to disk
- [ ] `summary` prints coverage percentages by tactic
- [ ] `export` produces a valid ATT&CK Navigator JSON layer
