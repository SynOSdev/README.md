## 📋 Mission Briefing

Deploying a firewall does not mean traffic is blocked. Enabling an EDR does not mean malware is caught. Security controls must be continuously validated to confirm they work as intended. In this lab you will test firewalls, IDS rules, and endpoint protections by sending controlled test traffic, attempting known bypasses, and measuring each control's real-world effectiveness.

## 📖 Background

Security controls are only as good as their configuration and maintenance. Drift, updates, and changing attack techniques mean controls degrade over time. Validation testing answers a simple question: "Does this control actually stop or detect what it is supposed to?"

Key concepts:
- **Control** — A technical safeguard (firewall, IDS, AV, proxy, WAF) designed to prevent or detect threats
- **Validation Test** — A controlled probe that exercises a specific control's function
- **Bypass** — A technique that circumvents a control without triggering an alert
- **Control Gap** — A scenario where no control exists to address a known threat
- **Compliance Validation** — Confirming controls meet regulatory or policy requirements

## 🔬 Lab Instructions

### Phase 1: Inventorying Security Controls

1. List firewall rules currently active on the host:
   ```bash
   sudo iptables -L -n -v 2>/dev/null || echo "iptables not available — check nftables"
   sudo nft list ruleset 2>/dev/null | head -30 || echo "Using iptables fallback"
   ```
2. Check if an IDS is running:
   ```bash
   pgrep -a suricata 2>/dev/null || echo "Suricata not running — install for full lab"
   ls /etc/suricata/rules/ 2>/dev/null | head -10
   ```
3. Check endpoint AV status:
   ```bash
   clamscan --version 2>/dev/null || echo "ClamAV not installed — install for full lab"
   ```
4. Document your control inventory:
   ```bash
   cat > ~/control_inventory.csv << 'EOF'
   control_id,control_type,tool,expected_behavior
   C001,network_firewall,iptables,Block inbound traffic on non-allowed ports
   C002,network_ids,suricata,Alert on known malicious signatures
   C003,endpoint_av,clamav,Detect known malware samples
   C004,host_firewall,iptables,Block outbound to known-bad IPs
   C005,file_integrity,auditd,Detect changes to critical system files
   EOF
   echo "Inventory saved: ~/control_inventory.csv"
   ```

### Phase 2: Designing and Running Validation Tests

1. Test the network firewall — attempt connections that should be blocked:
   ```bash
   # Test: outbound connection to a blocked port (assumes port 4444 is blocked)
   timeout 3 bash -c 'echo test | nc -w 2 127.0.0.1 4444' 2>/dev/null
   if [ $? -ne 0 ]; then
     echo "[PASS] C001 — Outbound to port 4444 blocked as expected"
   else
     echo "[FAIL] C001 — Connection to port 4444 succeeded — control gap!"
   fi
   ```
2. Test the IDS — send a request that should trigger a signature:
   ```bash
   # Trigger a test signature (Suricata ET Open test rule)
   curl -s --max-time 5 "http://testmynids.org/uid/index.html" -o /dev/null 2>/dev/null
   echo "Check Suricata logs: /var/log/suricata/fast.log"
   sudo tail -5 /var/log/suricata/fast.log 2>/dev/null || echo "[INFO] Suricata logs not available"
   ```
3. Test endpoint AV — scan a test file:
   ```bash
   # Create an EICAR test file (standard AV test string)
   echo 'X5O!P%@AP[4\PZX54(P^)7CC)7}$EICAR-STANDARD-ANTIVIRUS-TEST-FILE!$H+H*' > /tmp/eicar_test.txt
   clamscan /tmp/eicar_test.txt 2>/dev/null
   if [ $? -eq 1 ]; then
     echo "[PASS] C003 — ClamAV detected EICAR test file"
   else
     echo "[FAIL] C003 — ClamAV did not detect test file"
   fi
   rm -f /tmp/eicar_test.txt
   ```
4. Record results:
   ```bash
   cat > ~/validation_results.csv << 'EOF'
   control_id,test_description,result,notes
   C001,Outbound port 4444 block,PASS,Connection refused as expected
   C002,IDS signature trigger,PASS,Suricata alert generated
   C003,EICAR AV detection,PASS,ClamAV detected test file
   C004,Outbound to bad IP block,FAIL,Connection succeeded — rule missing
   C005,File integrity alert,PASS,Auditd logged modification
   EOF
   ```

### Phase 3: Bypass Testing and Gap Analysis

1. Attempt a simple firewall bypass using an allowed port:
   ```bash
   # If port 443 is allowed, test tunneling arbitrary data over it
   echo "[TEST] Tunneling data over port 443 to simulate bypass"
   timeout 3 bash -c 'echo "exfil_data" | nc -w 2 127.0.0.1 443' 2>/dev/null
   echo "[INFO] If data was sent, the firewall allows tunneling over HTTPS port"
   ```
2. Test IDS evasion with encoding:
   ```bash
   # Base64-encode a payload to test signature evasion
   echo "[TEST] Sending base64-encoded payload to evade plaintext signatures"
   echo "malicious_payload" | base64 | nc -w 2 127.0.0.1 80 2>/dev/null || true
   echo "[INFO] Check if IDS detected the encoded payload"
   ```
3. Summarize gaps:
   ```bash
   echo "=== Control Gap Summary ==="
   grep "FAIL" ~/validation_results.csv | while IFS=',' read -r cid desc result notes; do
     echo "  [GAP] $cid: $desc — $notes"
   done
   echo "=== Recommendations ==="
   echo "  1. Add outbound block rule for known-bad IP ranges"
   echo "  2. Add IDS rules for base64-encoded payloads"
   echo "  3. Schedule monthly control validation tests"
   ```

## 🛠️ Build Your Own

**Build Objective:** Create a custom security control validation suite that tests firewall, IDS, and endpoint controls

### Requirements
- Define controls and their expected behavior in a configuration file
- Run automated test cases against each control
- Record pass/fail results with timestamps
- Generate a validation report with gap analysis
- Support adding new controls and tests without code changes

### Starter Framework
```python
#!/usr/bin/env python3
"""GRIMOIRE Security Control Validator — LAB-028 Build Your Own"""

import json
import subprocess
import os
from datetime import datetime, timezone

CONFIG_FILE = os.path.expanduser("~/control_tests.json")
RESULTS_FILE = os.path.expanduser("~/validation_report.json")

DEFAULT_CONTROLS = [
    {
        "id": "C001",
        "name": "Firewall — Block port 4444 outbound",
        "test_cmd": "timeout 3 bash -c 'echo test | nc -w 2 127.0.0.1 4444' 2>/dev/null",
        "expect_fail": True,
        "description": "Outbound connection to port 4444 should be blocked",
    },
    {
        "id": "C002",
        "name": "IDS — Signature detection",
        "test_cmd": "curl -s --max-time 5 http://testmynids.org/uid/index.html -o /dev/null 2>/dev/null",
        "expect_fail": False,
        "log_check": "/var/log/suricata/fast.log",
        "description": "Suricata should alert on known test signature",
    },
    {
        "id": "C003",
        "name": "AV — EICAR detection",
        "test_cmd": "echo 'X5O!P%@AP[4\\PZX54(P^)7CC)7}$EICAR-STANDARD-ANTIVIRUS-TEST-FILE!$H+H*' > /tmp/eicar.txt && clamscan /tmp/eicar.txt 2>/dev/null; EXIT=$?; rm -f /tmp/eicar.txt; exit $EXIT",
        "expect_fail": True,
        "description": "ClamAV should detect EICAR test file (exit code 1)",
    },
]

def load_controls() -> list:
    """Load control definitions from config or use defaults."""
    if os.path.exists(CONFIG_FILE):
        with open(CONFIG_FILE) as f:
            return json.load(f)
    with open(CONFIG_FILE, "w") as f:
        json.dump(DEFAULT_CONTROLS, f, indent=2)
    return DEFAULT_CONTROLS

def run_test(control: dict) -> dict:
    """Execute a single control validation test."""
    ts = datetime.now(timezone.utc).isoformat()
    try:
        result = subprocess.run(
            control["test_cmd"], shell=True, timeout=15,
            capture_output=True, text=True,
        )
        cmd_failed = result.returncode != 0
    except subprocess.TimeoutExpired:
        cmd_failed = True

    if control.get("expect_fail"):
        passed = cmd_failed
    else:
        passed = not cmd_failed

    return {
        "control_id": control["id"],
        "control_name": control["name"],
        "timestamp": ts,
        "passed": passed,
        "details": control["description"],
    }

def generate_report(results: list):
    """Save and display the validation report."""
    total = len(results)
    passed = sum(1 for r in results if r["passed"])
    failed = total - passed

    report = {
        "generated_at": datetime.now(timezone.utc).isoformat(),
        "summary": {"total": total, "passed": passed, "failed": failed},
        "results": results,
    }
    with open(RESULTS_FILE, "w") as f:
        json.dump(report, f, indent=2)

    print(f"\n{'='*50}")
    print(f"  Control Validation Report")
    print(f"  Passed: {passed}/{total}  |  Failed: {failed}/{total}")
    print(f"{'='*50}")
    for r in results:
        status = "PASS" if r["passed"] else "FAIL"
        print(f"  [{status}] {r['control_id']}: {r['control_name']}")
    if failed:
        print(f"\n  Gaps Identified:")
        for r in results:
            if not r["passed"]:
                print(f"    - {r['control_id']}: {r['details']}")
    print(f"\n[+] Full report: {RESULTS_FILE}")

if __name__ == "__main__":
    controls = load_controls()
    print(f"[*] Running {len(controls)} control validation tests...\n")
    results = [run_test(c) for c in controls]
    generate_report(results)
```

### Validation
- [ ] Controls are loaded from a JSON configuration file
- [ ] Each test executes and returns a pass/fail result
- [ ] Report displays summary counts and per-control status
- [ ] Gaps are listed with remediation context
