## 📖 Background

Active scanning sends probes to target systems and analyzes their responses. This reveals which hosts are alive, which ports accept connections, what software is running behind those ports, and often what operating system the host uses. Enumeration goes deeper — extracting usernames, shares, group policies, and configuration details from discovered services.

Key concepts:
- **TCP Three-Way Handshake** — SYN → SYN-ACK → ACK; the basis of connect scans
- **SYN Scan (Half-Open)** — Sends SYN, reads response, never completes the handshake
- **UDP Scanning** — Connectionless; relies on ICMP unreachable or application responses
- **Service Fingerprinting** — Analyzing banners and probe responses to identify software
- **OS Fingerprinting** — Using TCP/IP stack behavior to determine the operating system

## 🔬 Lab Instructions

### Phase 1: Host Discovery

1. Start with an ARP scan on the local subnet to find live hosts:
   ```bash
   sudo nmap -sn -PR 10.10.10.0/24
   ```
2. Use ICMP echo, timestamp, and netmask requests for broader discovery:
   ```bash
   sudo nmap -sn -PE -PP -PM 10.10.10.0/24
   ```
3. Combine TCP SYN and ACK pings to bypass firewalls that block ICMP:
   ```bash
   sudo nmap -sn -PS22,80,443 -PA80,443 10.10.10.0/24
   ```
4. Record all live hosts with IP and MAC addresses in your target list

### Phase 2: Port Scanning

1. Run a SYN scan on common ports against discovered hosts:
   ```bash
   sudo nmap -sS -top-ports 1000 -T4 10.10.10.50
   ```
2. Perform a full 65535-port scan on high-value targets:
   ```bash
   sudo nmap -sS -p- -T4 --min-rate 1000 10.10.10.50
   ```
3. Scan for common UDP services:
   ```bash
   sudo nmap -sU --top-ports 50 -T4 10.10.10.50
   ```
4. Use masscan for rapid large-network sweeps:
   ```bash
   sudo masscan 10.10.10.0/24 -p1-65535 --rate 1000 -oG masscan_results.gnmap
   ```
5. Compare masscan and nmap results to ensure nothing was missed

### Phase 3: Service and OS Enumeration

1. Run version detection and OS fingerprinting:
   ```bash
   sudo nmap -sV -O -sC -p <open_ports> 10.10.10.50
   ```
2. Use nmap scripting engine (NSE) for deeper enumeration:
   ```bash
   sudo nmap --script=banner,vuln -p <open_ports> 10.10.10.50
   ```
3. Enumerate HTTP services for server headers and technologies:
   ```bash
   sudo nmap --script=http-title,http-headers,http-methods -p 80,443 10.10.10.50
   ```
4. Record all service versions, OS guesses, and script output

### Phase 4: Protocol-Specific Enumeration

1. Enumerate SMB shares and users with enum4linux:
   ```bash
   enum4linux -a 10.10.10.50
   ```
2. Scan NetBIOS names across the subnet:
   ```bash
   nbtscan 10.10.10.0/24
   ```
3. Walk the SNMP tree if community strings are known:
   ```bash
   snmpwalk -v2c -c public 10.10.10.50
   ```
4. Enumerate LDAP if port 389/636 is open:
   ```bash
   ldapsearch -x -H ldap://10.10.10.50 -b "dc=grimoire,dc=lab"
   ```
5. Compile all enumeration data into a service-level target map

### Phase 5: Scan Evasion Basics

1. Experiment with timing templates to reduce scan footprint:
   ```bash
   sudo nmap -sS -T2 -p 80,443 10.10.10.50
   ```
2. Fragment packets to evade simple IDS signatures:
   ```bash
   sudo nmap -f -sS -p 80 10.10.10.50
   ```
3. Use decoy addresses to obscure the scan origin:
   ```bash
   sudo nmap -D RND:5 -sS -p 80 10.10.10.50
   ```
4. Randomize scan order and use source port spoofing:
   ```bash
   sudo nmap --randomize-hosts -g 53 -sS -p- 10.10.10.50
   ```
