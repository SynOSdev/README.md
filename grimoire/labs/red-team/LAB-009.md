## 📖 Background

Reconnaissance is the first phase of the cyber kill chain. Passive recon gathers intelligence without directly interacting with the target, making it virtually undetectable. OSINT leverages publicly available sources — search engines, DNS records, social media, certificate transparency logs, and data breach repositories — to build a picture of an organization's attack surface.

Key concepts:
- **Passive vs. Active Recon** — Passive never touches the target; active sends probes
- **OSINT Cycle** — Planning, collection, processing, analysis, dissemination
- **Attack Surface** — All externally reachable assets: domains, IPs, services, people
- **Google Dorking** — Using advanced operators (`site:`, `filetype:`, `intitle:`, `inurl:`) to find exposed data
- **DNS Enumeration** — Discovering subdomains, mail servers, and name servers from public records

## 🔬 Lab Instructions

### Phase 1: Domain and DNS Intelligence

1. Choose a target domain from the GRIMOIRE practice range (e.g., `grimoire-target.lab`)
2. Run a WHOIS lookup to identify registrant, name servers, and creation dates:
   ```bash
   whois grimoire-target.lab
   ```
3. Enumerate DNS records using `dig` and `host`:
   ```bash
   dig grimoire-target.lab ANY +noall +answer
   dig mx grimoire-target.lab
   dig ns grimoire-target.lab
   host -t txt grimoire-target.lab
   ```
4. Attempt a DNS zone transfer (often blocked but always worth trying):
   ```bash
   dig axfr grimoire-target.lab @ns1.grimoire-target.lab
   ```
5. Document all discovered domains, subdomains, IPs, and mail servers

### Phase 2: Search Engine Dorking

1. Use Google Dork operators to discover exposed resources:
   ```
   site:grimoire-target.lab filetype:pdf
   site:grimoire-target.lab intitle:"index of"
   site:grimoire-target.lab inurl:admin
   site:grimoire-target.lab ext:sql | ext:bak | ext:log
   ```
2. Search for exposed configuration files and credentials:
   ```
   site:grimoire-target.lab filetype:env
   site:grimoire-target.lab "password" filetype:txt
   ```
3. Check paste sites and code repositories for leaked data
4. Record all findings with URLs and descriptions

### Phase 3: Automated Harvesting

1. Run theHarvester to collect emails and subdomains:
   ```bash
   theHarvester -d grimoire-target.lab -b google,bing,dnsdumpster -l 200
   ```
2. Launch recon-ng and load appropriate modules:
   ```bash
   recon-ng
   marketplace install all
   workspaces create grimoire-target
   modules load recon/domains-hosts/hackertarget
   options set SOURCE grimoire-target.lab
   run
   ```
3. Query Shodan for internet-facing services:
   ```bash
   shodan search hostname:grimoire-target.lab
   shodan host <discovered_ip>
   ```
4. Cross-reference results from all tools to build a unified target profile

### Phase 4: Social Media and People Intelligence

1. Identify key personnel through LinkedIn, company pages, and public directories
2. Map organizational structure and identify potential phishing targets
3. Search for employee email patterns (first.last, flast, etc.)
4. Check data breach databases (e.g., Have I Been Pwned API) for compromised credentials
5. Compile a people-focused intelligence report

### Phase 5: Report Compilation

1. Consolidate all findings into a structured markdown report
2. Categorize by asset type: domains, IPs, services, people, credentials
3. Assign risk ratings to each finding
4. Identify the most promising attack vectors based on recon data
