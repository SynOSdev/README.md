## 📖 Background

Digital forensics follows a strict methodology to ensure evidence is admissible and reliable. The core principle is that evidence must never be modified — analysts work on copies, verify integrity with hashes, and document every action. Timeline analysis reconstructs the sequence of events by correlating timestamps from multiple sources.

Key concepts:
- **Forensic image** — Bit-for-bit copy of a storage device
- **Write blocker** — Hardware or software preventing writes to evidence media
- **File carving** — Recovering files based on headers and footers, not filesystem metadata
- **MAC timestamps** — Modified, Accessed, Changed times on filesystem objects
- **Chain of custody** — Documentation tracking who handled evidence and when
- **Slack space** — Unused space in allocated disk clusters that may contain remnant data

## 🔬 Lab Instructions

### Phase 1: Forensic Imaging

1. Create a test evidence disk for the lab:
   ```bash
   mkdir -p ~/grimoire/forensics-lab/{evidence,images,reports,timeline}
   # Create a small test filesystem
   dd if=/dev/zero of=/tmp/evidence_disk.img bs=1M count=50
   mkfs.ext4 /tmp/evidence_disk.img
   ```
2. Mount and populate the test disk with artifacts:
   ```bash
   sudo mkdir -p /mnt/evidence
   sudo mount -o loop /tmp/evidence_disk.img /mnt/evidence
   sudo bash -c 'echo "Sensitive document contents" > /mnt/evidence/secret.txt'
   sudo bash -c 'echo "192.168.1.50 - attacker accessed at 03:14" > /mnt/evidence/access.log'
   sudo bash -c 'echo "#!/bin/bash" > /mnt/evidence/backdoor.sh'
   # Delete some files to practice recovery
   sudo rm /mnt/evidence/secret.txt
   sudo umount /mnt/evidence
   ```
3. Create a forensic image with hash verification:
   ```bash
   # Image the disk with SHA-256 verification
   sha256sum /tmp/evidence_disk.img > ~/grimoire/forensics-lab/evidence/original_hash.txt
   dd if=/tmp/evidence_disk.img of=~/grimoire/forensics-lab/images/forensic_copy.img bs=4096
   sha256sum ~/grimoire/forensics-lab/images/forensic_copy.img > ~/grimoire/forensics-lab/evidence/copy_hash.txt
   # Verify hashes match
   diff <(cut -d' ' -f1 ~/grimoire/forensics-lab/evidence/original_hash.txt) \
        <(cut -d' ' -f1 ~/grimoire/forensics-lab/evidence/copy_hash.txt)
   ```
4. Document the chain of custody: date, time, examiner, hash values, storage location

### Phase 2: File Recovery and Artifact Extraction

1. Install forensic tools: `sudo apt-get install foremost sleuthkit exiftool -y`
2. Attempt file recovery using foremost:
   ```bash
   foremost -i ~/grimoire/forensics-lab/images/forensic_copy.img -o ~/grimoire/forensics-lab/evidence/recovered/
   ls -la ~/grimoire/forensics-lab/evidence/recovered/
   ```
3. Use The Sleuth Kit to examine the filesystem:
   ```bash
   # List filesystem information
   fsstat ~/grimoire/forensics-lab/images/forensic_copy.img
   # List all files including deleted ones
   fls -r -d ~/grimoire/forensics-lab/images/forensic_copy.img
   ```
4. Extract a specific file by inode number:
   ```bash
   # Find the inode of a deleted file from fls output, then extract
   # icat forensic_copy.img <inode_number> > recovered_file.txt
   ```
5. Extract metadata timestamps from recovered files:
   ```bash
   stat ~/grimoire/forensics-lab/evidence/recovered/*
   exiftool ~/grimoire/forensics-lab/evidence/recovered/*
   ```

### Phase 3: Timeline Analysis

1. Generate a filesystem timeline using The Sleuth Kit:
   ```bash
   # Create a bodyfile from the forensic image
   fls -r -m "/" ~/grimoire/forensics-lab/images/forensic_copy.img > ~/grimoire/forensics-lab/timeline/bodyfile.txt
   # Convert bodyfile to a human-readable timeline
   mactime -b ~/grimoire/forensics-lab/timeline/bodyfile.txt > ~/grimoire/forensics-lab/timeline/timeline.csv
   ```
2. Analyze the timeline for suspicious activity:
   ```bash
   # Look for files created or modified during suspicious time windows
   grep "backdoor" ~/grimoire/forensics-lab/timeline/timeline.csv
   # Sort by timestamp to see sequence of events
   sort -t',' -k1 ~/grimoire/forensics-lab/timeline/timeline.csv | head -30
   ```
3. Correlate filesystem timestamps with log entries
4. Identify the attack sequence: initial access, file drops, data access, cleanup
5. Write a forensic examination report documenting:
   - Evidence items and their hashes
   - Recovery methods used and results
   - Timeline of events reconstructed from artifacts
   - Key findings and conclusions
