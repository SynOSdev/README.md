## 📖 Background

Malware analysis is the process of understanding the behavior and purpose of malicious software. Analysts use two primary approaches: static analysis examines the file without running it, while dynamic analysis observes its behavior during execution in a controlled environment.

Key concepts:
- **Static analysis** — Examining file properties, strings, structure, and code without execution
- **Dynamic analysis** — Running malware in an isolated sandbox and observing behavior
- **Indicators of Compromise (IOCs)** — Artifacts that identify malicious activity (hashes, IPs, domains, file paths)
- **YARA rules** — Pattern-matching language for classifying and detecting malware
- **Sandbox** — Isolated environment for safely executing and observing malware
- **Triage** — Rapid initial assessment to prioritize analysis effort

## 🔬 Lab Instructions

### Phase 1: File Identification and Static Analysis

1. Create a safe sample file for analysis practice:
   ```bash
   mkdir -p ~/grimoire/malware-lab/samples ~/grimoire/malware-lab/reports
   # Create a benign test file that mimics suspicious characteristics
   echo '#!/bin/bash' > ~/grimoire/malware-lab/samples/sample001.dat
   echo '# This is a safe lab sample — not actual malware' >> ~/grimoire/malware-lab/samples/sample001.dat
   echo 'curl http://example.com/callback' >> ~/grimoire/malware-lab/samples/sample001.dat
   echo 'echo "base64encodedpayload" | base64 -d > /tmp/.hidden' >> ~/grimoire/malware-lab/samples/sample001.dat
   ```
2. Identify the file type:
   ```bash
   file ~/grimoire/malware-lab/samples/sample001.dat
   xxd ~/grimoire/malware-lab/samples/sample001.dat | head -5
   ```
3. Generate file hashes for identification:
   ```bash
   md5sum ~/grimoire/malware-lab/samples/sample001.dat
   sha256sum ~/grimoire/malware-lab/samples/sample001.dat
   ```
4. Extract strings and look for IOCs:
   ```bash
   strings ~/grimoire/malware-lab/samples/sample001.dat
   strings -n 8 ~/grimoire/malware-lab/samples/sample001.dat | grep -iE '(http|ftp|\.com|\.exe|/tmp|base64)'
   ```
5. Check metadata with exiftool: `exiftool ~/grimoire/malware-lab/samples/sample001.dat`

### Phase 2: YARA Rule Development

1. Install YARA: `sudo apt-get install yara -y`
2. Write a YARA rule to detect the sample's characteristics:
   ```yara
   rule GRIMOIRE_Lab_Sample {
       meta:
           description = "Detects lab sample with callback and encoded payload"
           author = "GRIMOIRE Student"
           date = "2024-01-01"
       strings:
           $callback = "curl" ascii
           $hidden_file = "/tmp/.hidden" ascii
           $encoding = "base64" ascii
           $shebang = "#!/bin/bash" ascii
       condition:
           $shebang at 0 and ($callback or $hidden_file) and $encoding
   }
   ```
3. Save the rule to `~/grimoire/malware-lab/rules/lab_rules.yar`
4. Test the rule against your sample:
   ```bash
   yara ~/grimoire/malware-lab/rules/lab_rules.yar ~/grimoire/malware-lab/samples/
   ```
5. Refine the rule — test against benign files to verify no false positives

### Phase 3: Behavioral Analysis and Triage

1. Use `strace` to trace system calls of a safe script:
   ```bash
   strace -f -e trace=network,file -o /tmp/strace_output.txt bash ~/grimoire/malware-lab/samples/sample001.dat
   ```
2. Analyze the strace output for suspicious behavior:
   ```bash
   grep -E '(connect|open|write|exec)' /tmp/strace_output.txt
   ```
3. Identify behavioral indicators:
   - Network connections (callback URLs, C2 addresses)
   - File system modifications (dropped files, persistence mechanisms)
   - Process creation (child processes, privilege escalation attempts)
4. Build a triage checklist:
   - File type and size
   - Hash values (SHA-256, MD5)
   - Embedded strings and URLs
   - YARA rule matches
   - Behavioral indicators from sandbox
5. Generate a triage report documenting all findings

def compute_hashes(filepath):
    """Compute MD5, SHA-1, and SHA-256 hashes of a file."""
    hashes = {}
    with open(filepath, "rb") as f:
        data = f.read()
    hashes["md5"] = hashlib.md5(data).hexdigest()
    hashes["sha1"] = hashlib.sha1(data).hexdigest()
    hashes["sha256"] = hashlib.sha256(data).hexdigest()
    hashes["size_bytes"] = len(data)
    return hashes

def identify_filetype(filepath):
    """Use the file command to identify the file type."""
    try:

            ["file", "--brief", filepath],

        )
        return result.stdout.strip()
    except (subprocess.SubprocessError, FileNotFoundError):
        return "unknown"
