## 📖 Background

Firewalls are the first line of defense in network security. They examine packets against a set of rules and decide whether to allow, drop, or reject traffic. Modern firewalls use stateful inspection to track connection states, enabling rules that permit return traffic for established connections while blocking unsolicited inbound packets.

Key concepts:
- **Default deny** — Block everything, then explicitly allow only what is needed
- **Chains** — INPUT (to host), OUTPUT (from host), FORWARD (routed through)
- **Stateful inspection** — Tracking connection state (NEW, ESTABLISHED, RELATED)
- **Zones** — Logical groupings of interfaces with different trust levels
- **Network segmentation** — Isolating network segments to limit lateral movement

## 🔬 Lab Instructions

### Phase 1: iptables Fundamentals

1. List current iptables rules: `sudo iptables -L -n -v --line-numbers`
2. Set default policies to DROP for INPUT and FORWARD chains:
   ```bash
   sudo iptables -P INPUT DROP
   sudo iptables -P FORWARD DROP
   sudo iptables -P OUTPUT ACCEPT
   ```
3. Allow loopback traffic:
   ```bash
   sudo iptables -A INPUT -i lo -j ACCEPT
   ```
4. Allow established and related connections:
   ```bash
   sudo iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
   ```
5. Allow SSH from a specific subnet:
   ```bash
   sudo iptables -A INPUT -p tcp --dport 22 -s 192.168.1.0/24 -j ACCEPT
   ```
6. Log and drop all other traffic:
   ```bash
   sudo iptables -A INPUT -j LOG --log-prefix "DROPPED: " --log-level 4
   sudo iptables -A INPUT -j DROP
   ```

### Phase 2: nftables and Zone Configuration

1. Create an nftables ruleset equivalent to Phase 1:
   ```bash
   sudo nft add table inet filter
   sudo nft add chain inet filter input { type filter hook input priority 0 \; policy drop \; }
   sudo nft add rule inet filter input iif lo accept
   sudo nft add rule inet filter input ct state established,related accept
   sudo nft add rule inet filter input tcp dport 22 ip saddr 192.168.1.0/24 accept
   ```
2. Configure ufw as a simplified alternative:
   ```bash
   sudo ufw default deny incoming
   sudo ufw default allow outgoing
   sudo ufw allow from 192.168.1.0/24 to any port 22
   sudo ufw enable
   ```
3. Explore firewalld zones — assign interfaces to zones with different trust levels
4. Create a custom zone for a DMZ segment with limited allowed services

### Phase 3: Network Segmentation and Verification

1. Design a three-zone architecture: trusted (LAN), DMZ, untrusted (WAN)
2. Write FORWARD chain rules that allow DMZ to serve HTTP/HTTPS to the WAN
3. Block direct WAN-to-LAN traffic while allowing LAN-to-WAN
4. Use `nmap` from each zone to verify port accessibility
5. Capture packets with `tcpdump` to confirm rules are applied correctly
6. Save your ruleset for persistence:
   ```bash
   sudo iptables-save > /etc/iptables/rules.v4
   ```
