## 📖 Background

Intrusion detection fills the gap between firewall filtering and full traffic analysis. An IDS passively monitors network traffic, comparing it against known attack signatures or behavioral baselines. An IPS sits inline and can block malicious traffic in real-time.

Key concepts:
- **Signature-based detection** — Matching traffic against known attack patterns
- **Anomaly-based detection** — Flagging deviations from normal baseline behavior
- **Alert fatigue** — The operational cost of too many false positives
- **Rule tuning** — Adjusting thresholds, suppressing noise, refining signatures
- **Evasion techniques** — Fragmentation, encoding, and protocol tricks attackers use to bypass IDS

## 🔬 Lab Instructions

### Phase 1: IDS Deployment

1. Install Suricata on your system:
   ```bash
   sudo apt-get install suricata -y
   ```
2. Configure the HOME_NET variable in `/etc/suricata/suricata.yaml`:
   ```yaml
   vars:
     address-groups:
       HOME_NET: "[192.168.1.0/24]"
       EXTERNAL_NET: "!$HOME_NET"
   ```
3. Update the default rule sets:
   ```bash
   sudo suricata-update
   ```
4. Start Suricata in IDS mode on your primary interface:
   ```bash
   sudo suricata -c /etc/suricata/suricata.yaml -i eth0
   ```
5. Verify alerts are being logged to `/var/log/suricata/fast.log`

### Phase 2: Custom Rule Writing

1. Understand the Suricata rule format:
   ```
   action protocol src_ip src_port -> dst_ip dst_port (options;)
   ```
2. Write a rule to detect ICMP ping sweeps:
   ```
   alert icmp $EXTERNAL_NET any -> $HOME_NET any (msg:"GRIMOIRE - ICMP Ping Sweep Detected"; itype:8; threshold:type threshold, track by_src, count 5, seconds 10; sid:1000001; rev:1;)
   ```
3. Write a rule to detect SSH brute force attempts:
   ```
   alert tcp $EXTERNAL_NET any -> $HOME_NET 22 (msg:"GRIMOIRE - SSH Brute Force Attempt"; flow:to_server,established; threshold:type threshold, track by_src, count 5, seconds 60; sid:1000002; rev:1;)
   ```
4. Write a rule to detect suspicious HTTP User-Agent strings:
   ```
   alert http $EXTERNAL_NET any -> $HOME_NET any (msg:"GRIMOIRE - Suspicious User-Agent"; http.user_agent; content:"nikto"; nocase; sid:1000003; rev:1;)
   ```
5. Load your custom rules and test them with `tcpreplay` against sample captures

### Phase 3: Alert Analysis and Tuning

1. Generate test traffic to trigger your rules:
   ```bash
   # Trigger ICMP rule
   ping -c 10 -i 0.1 <target_ip>
   # Trigger HTTP rule
   curl -A "nikto" http://<target_ip>/
   ```
2. Review alerts in `/var/log/suricata/fast.log` and `eve.json`
3. Parse the EVE JSON log to extract alert summaries:
   ```bash
   cat /var/log/suricata/eve.json | jq 'select(.event_type=="alert") | {timestamp, src_ip: .src_ip, dest_ip: .dest_ip, msg: .alert.signature}'
   ```
4. Suppress noisy rules that generate false positives:
   ```yaml
   # In threshold.config
   suppress gen_id 1, sig_id 2100498, track by_src, ip 192.168.1.100
   ```
5. Tune thresholds to balance detection sensitivity with alert volume
6. Document your tuning decisions and rationale
