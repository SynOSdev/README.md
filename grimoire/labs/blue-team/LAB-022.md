## 📖 Background

EDR solutions provide continuous monitoring and response capabilities at the endpoint level. Unlike antivirus which relies primarily on file signatures, EDR tracks process behavior, system calls, network connections, and file operations to detect threats that evade traditional defenses. The key advantage is visibility into the full attack chain on the host.

Key concepts:
- **Telemetry** — Raw data collected from endpoints (process events, file changes, network connections)
- **Behavioral detection** — Identifying threats by what they do, not what they look like
- **File integrity monitoring (FIM)** — Detecting unauthorized changes to critical files
- **Process lineage** — Tracking parent-child process relationships to spot anomalies
- **Response actions** — Automated or manual containment at the endpoint level (kill process, isolate host, quarantine file)

## 🔬 Lab Instructions

### Phase 1: Process Monitoring with osquery

1. Install osquery:
   ```bash
   sudo apt-get install osquery -y 2>/dev/null || echo "Install osquery from https://osquery.io"
   ```
2. Use osquery interactive mode to explore endpoint state:
   ```bash
   osqueryi
   # List running processes
   SELECT pid, name, cmdline, uid, start_time FROM processes ORDER BY start_time DESC LIMIT 20;
   # Find processes listening on network ports
   SELECT p.pid, p.name, l.port, l.address FROM listening_ports l JOIN processes p ON l.pid = p.pid;
   # Check for unusual scheduled tasks
   SELECT * FROM crontab;
   # List loaded kernel modules
   SELECT name, size, status FROM kernel_modules WHERE status = 'Live';
   ```
3. Create an osquery pack for continuous monitoring:
   ```json
   {
     "queries": {
       "new_processes": {
         "query": "SELECT pid, name, cmdline, parent FROM processes WHERE start_time > (strftime('%s','now') - 60);",
         "interval": 60,
         "description": "New processes started in the last minute"
       },
       "listening_ports": {
         "query": "SELECT l.port, l.address, p.name FROM listening_ports l JOIN processes p ON l.pid = p.pid;",
         "interval": 300,
         "description": "Processes listening on network ports"
       }
     }
   }
   ```
4. Query for suspicious patterns:
   ```sql
   -- Processes running from /tmp or /dev/shm
   SELECT pid, name, path, cmdline FROM processes WHERE path LIKE '/tmp/%' OR path LIKE '/dev/shm/%';
   -- Processes with deleted binary on disk
   SELECT pid, name, path FROM processes WHERE on_disk = 0;
   ```

### Phase 2: File Integrity Monitoring

1. Set up auditd to monitor critical paths:
   ```bash
   sudo apt-get install auditd -y
   # Monitor /etc/passwd for modifications
   sudo auditctl -w /etc/passwd -p wa -k passwd_changes
   # Monitor /etc/shadow
   sudo auditctl -w /etc/shadow -p wa -k shadow_changes
   # Monitor SSH configuration
   sudo auditctl -w /etc/ssh/sshd_config -p wa -k sshd_config
   # Monitor cron directories
   sudo auditctl -w /etc/cron.d/ -p wa -k cron_changes
   ```
2. Use inotifywait for real-time file monitoring:
   ```bash
   # Watch for changes in /etc in the background
   inotifywait -m -r -e modify,create,delete /etc/ 2>/dev/null &
   ```
3. Create a baseline of critical file hashes:
   ```bash
   find /etc -type f -name "*.conf" -exec sha256sum {} \; > ~/grimoire/edr-lab/baseline_hashes.txt
   ```
4. Build a script that compares current hashes against the baseline to detect changes
5. Test by modifying a monitored file and observing the alert

### Phase 3: Behavioral Detection and Response

1. Define detection rules for common attack patterns:
   - **Reverse shell:** Process with established outbound connection spawning `/bin/sh` or `/bin/bash`
   - **Privilege escalation:** Non-root process spawning a root-owned child
   - **Persistence:** New files created in `/etc/cron.d/`, `/etc/systemd/system/`, or user crontab changes
   - **Data staging:** Large files created in `/tmp/` or compressed archives created
2. Monitor for reverse shell indicators:
   ```bash
   # Check for shells with network connections
   lsof -i -nP | grep -E '(bash|sh|nc|ncat|python|perl|ruby)' | grep ESTABLISHED
   ```
3. Build a detection script that checks for these patterns on a schedule
4. Implement response actions:
   ```bash
   # Kill a suspicious process
   kill -9 <pid>
   # Block outbound connection to suspicious IP
   sudo iptables -A OUTPUT -d <suspicious_ip> -j DROP
   ```
5. Log all detections and responses with timestamps for investigation
